# LLM работа и составление промптов
**Дата извлечения**: 2026-05-16
**Источник**: 358f1be6
**Раздел**: system-prompts

## Суть
Чат представляет собой глубокое руководство по промпт-инжинирингу, объединяющее понимание внутренней архитектуры LLM (Transformer, Self-Attention) с практическими техниками составления промптов. Особый акцент сделан на создании структурированных «мета-промптов» для генерации промптов и адаптации под конкретные модели (DeepSeek V4) и сценарии (LLM как мастер ADnD-игры).

## Ключевые тезисы
- **LLM — вероятностный «предсказатель»**: модель предсказывает следующее слово (токен) на основе контекста; промптинг — искусство создания контекста, ведущего к нужному предсказанию.
- **Архитектура Transformer и Self-Attention**: ключевая инновация — параллельный анализ всех слов в последовательности с взвешиванием их взаимной важности; существуют Encoder-only, Decoder-only и Encoder-Decoder модели.
- **Трёхэтапное обучение**: предобучение (pre-training) → тонкая настройка инструкций (instruction tuning) → обучение с подкреплением от человека (RLHF).
- **Фундаментальные принципы промптинга**: ясность и конкретность, задание роли (Persona), декомпозиция сложных задач на цепочку простых промптов.
- **Продвинутые техники**: Zero-shot / Few-shot (задание шаблонов), Chain-of-Thought (пошаговое рассуждение), структурирование промпта и вывода (явные разделители, указание формата).
- **Железное правило порядка**: инструкции и системные правила должны предшествовать пользовательскому вводу; модель читает слева направо, и ранний контекст задаёт «мировоззрение» для генерации.
- **Архитектура мета-промпта**: роль → правила → формат вывода → контекст → разделитель → реплика/задача пользователя; такой шаблон самодостаточен и может быть вставлен в новое окно чата.
- **Адаптация под DeepSeek**: избегать System Prompt для R1, использовать чёткие Markdown-заголовки/XML-теги, минимизировать few-shot для R1, поддерживать температуру 0.5–0.7, требовать проактивного уточнения перед генерацией.
