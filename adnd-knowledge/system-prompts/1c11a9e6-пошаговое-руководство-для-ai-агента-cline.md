# Пошаговое руководство для AI-агента Cline
**Дата извлечения**: 2026-05-16
**Источник**: 1c11a9e6
**Раздел**: system-prompts, architecture

## Суть
Чат содержит переработанное техническое задание (ТЗ) в виде пошагового руководства для AI-агента Cline по реализации MVP мультиплеерной RPG-системы «Экс-Ис: Архитектор Миров». Документ разбит на 9 этапов (0–8), каждый с конкретными артефактами (файлы, код, конфигурации) и чек-листами, охватывающими бэкенд на FastAPI/WebSocket, графовую БД Neo4j, векторную ChromaDB, интеграцию с DeepSeek API, игровой цикл и минимальный SPA-клиент.

## Ключевые тезисы
- **Структура этапов строго фиксирована**: инициализация (0) → модели данных (1) → Neo4j (2) → ChromaDB (3) → API/WebSocket (4) → игровой цикл (5) → промпты и LLM (6) → фронтенд (7) → тестирование и docker-compose (8).
- **Детальные спецификации для каждого этапа**: точные имена файлов, Pydantic-схемы, Cypher-запросы для индексов/констрейнтов, схемы метаданных ChromaDB, JSON-примеры REST и WebSocket сообщений.
- **Игровой цикл с двумя вызовами LLM**: первый вызов генерирует нарратив и опциональные броски D20; второй (после получения результатов) – финальный нарратив; максимум 2 вызова на ход.
- **Обработка скрытых бросков**: сервер не передаёт DC игроку, но передаёт модели фактический успех/провал с пометкой «скрытый» для корректного нарративного описания.
- **Жёсткие правила парсинга XML**: ответы LLM содержат строгие теги (<narrative>, <dice_request>, <service_update> и др.), которые сервер извлекает для обновления мира (Neo4j + ChromaDB).
- **Промпты Экс-Ис и Анимы**: полные тексты из ТЗ с плейсхолдерами {} для подстановки данных сессии, включены в этап 6 как константы.
- **Интеграционное тестирование**: в этапе 8 предусмотрен автоматизированный тест полного цикла с docker-compose, моками LLM и проверкой состояния БД.

## Связи
- Связано с: **system-prompts** (этап 6 содержит полные тексты системных промптов для Экс-Ис и Анимы), **architecture** (общая архитектура: FastAPI, Neo4j, ChromaDB, WebSocket, DeepSeek API), **db-schema** (этап 2: Neo4j constraints/indices), **api-spec** (этап 4: REST-эндпоинты и WebSocket-сообщения), **game-loop** (этап 5: логика оркестратора), **frontend-spec** (этап 7: минимальный SPA).
