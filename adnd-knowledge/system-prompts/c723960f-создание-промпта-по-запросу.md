# Создание промпта по запросу
**Дата извлечения**: 2026-05-16
**Источник**: c723960f
**Раздел**: system-prompts

## Суть
Чат демонстрирует методологию создания эффективных промптов для языковых моделей, где эксперт последовательно уточняет задачу пользователя, структурирует промпт по строгим правилам (роль, задача, контекст, формат, ограничения) и обеспечивает его самодостаточность. Основной фокус — превращение сырых идей и технических заданий в готовые к использованию промпты для AI-агентов (Cline) и LLM.

## Ключевые тезисы
- **Правила генерации промптов**: Перед созданием промпта обязательно задавать уточняющие вопросы (не более 3) для определения цели, аудитории, формата, стиля и ограничений. Сам промпт должен содержать чёткие секции: Роль, Задача, Контекст, Формат, Ограничения.
- **Самодостаточность и конкретность**: Каждый созданный промпт должен быть готов к копированию и вставке, содержать однозначные инструкции, избегать размытых фраз и включать просьбу «Если информации недостаточно, скажи об этом».
- **Адаптация под среду выполнения**: При создании промптов для Cline (AI-агент в VSCode) необходимо учитывать порядок обработки правил (`.clinerules`), режимы Plan/Act и возможность интеграции с внешними инструментами (MCP-серверы).
- **Итеративное улучшение через диалог**: Эффективный промпт создаётся через серию уточнений, где эксперт анализирует ответы пользователя, при необходимости ищет дополнительную информацию (документация Cline, архитектуры AI-DM) и корректирует структуру промпта.
- **Применение в реальных проектах**: Промпты создавались для аудита кодовой базы (игра Exis), проверки и переработки технических заданий (подсистема «Вечный Архив»), анализа UI/UX и сравнения с существующими решениями (AI-DM системы из научных работ и GitHub).
- **Разделение на этапы в сложных задачах**: Для задач, требующих сбора внешней информации и последующего анализа, рекомендуется разделять процесс на два промпта: сначала сбор данных (с возможностью запроса ссылок), затем аудит и генерация итогового документа.
