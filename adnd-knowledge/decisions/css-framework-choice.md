# Выбор CSS-фреймворка: Bootstrap vs Tailwind CSS (2026)

**Дата извлечения**: 2026-05-16
**Источник**: 2026-04-21/bf276322-выбор-css-фреймворка-в-2026.md
**Раздел**: decisions

## Суть
Сравнение подходов к CSS в 2026 году. Tailwind CSS (утилитарный) стал стандартом для новых проектов, Bootstrap (компонентный) остаётся лидером по числу сайтов и хорош для MVP/админок.

## Ключевые факты

### Bootstrap
- Компонентный подход: готовые блоки (`btn`, `card`, `navbar`)
- Низкий порог входа — прототип за пару часов
- Большой CSS-бандл (~150 КБ) — все стили грузятся, даже неиспользуемые
- Интеграция с React/Vue через прослойки (`react-bootstrap`)
- ~70% рынка по числу сайтов, 172K+ звёзд на GitHub
- Удовлетворённость разработчиков: 55%
- **Сценарий**: быстрые прототипы, MVP, админ-панели, поддержка старых проектов

### Tailwind CSS
- Утилитарный подход: атомарные классы (`flex`, `pt-4`, `text-center`) в HTML
- Высокая гибкость — уникальный дизайн под полным контролем
- Маленький бандл (6-30 КБ) — JIT-компилятор включает только используемые стили
- Нативная интеграция с React, Next.js, Vue, Svelte
- 8M+ еженедельных загрузок NPM
- Удовлетворённость разработчиков: 81%
- **Сценарий**: современные проекты, стартапы, SaaS, SPA с кастомным дизайном

### Альтернативы
- **shadcn/ui** (2026): коллекция компонентов для React на Tailwind — копируешь в проект. Лучшее для быстрой разработки с уникальным дизайном.
- **Material UI (MUI)**: для строгого Google Material Design. Тяжёлая, навязывает дизайн.
- **Bulma**: чистый CSS без JS, элегантные компоненты, но без встроенных интерактивных элементов.
- **Foundation**: потерял популярность к 2026.
- **Pure.css**: микро-фреймворк для базовых вещей (сетка + кнопки).
