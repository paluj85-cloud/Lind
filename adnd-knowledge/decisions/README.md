## Архитектура и инфраструктура

### Выбор архитектуры
- Принята **Single-Pass архитектура** (монолитный промпт) вместо Multi-Agent подхода. Это осознанное решение, направленное на упрощение логики и повышение согласованности ответов.
- Архитектура предполагает разделение LLM (Мастер) и внешней памяти «Вечный Архив». Это обеспечивает чистоту контекстного окна, бесконечную память без деградации, устойчивость к галлюцинациям и возможность масштабирования на мультиплеер.

### Выбор AI-сервисов
- **Основная LLM:** DeepSeek.
- **Генерация изображений:** YandexArt.
- **Распознавание голоса:** SaluteSpeech (приложение постоянно слушает игроков в реальном времени).

### База данных и хранение
- Начальное решение — **SQLite + FTS5** (встроенная БД, без внешних СУБД).
- Позднее признана необходимость **векторного поиска**. Рассматриваются варианты: `sqlite-vss` или переход на **PostgreSQL с pgvector**.

### Инициализация мира
- Скрипт `scripts/init_world.py` создаёт стартовый мир: локации, NPC, предметы, квесты и факты в Neo4j и ChromaDB.
- **Идемпотентность:** скрипт можно запускать многократно без побочных эффектов.
- **Триггер запуска:** скрипт вызывается один раз перед запуском, а также автоматически при старте сессии, если стартовая локация отсутствует.

### Аутентификация и пользователи
- Решение **не делать регистрацию**. При запуске приложения пользователь может выбрать существующего или создать нового пользователя.

## Промпт-инжиниринг и логика ИИ-Мастера

### Эволюция промпта
- **Версия 2.0:** Создана расширенная версия системного промпта (ядро расширено минимум в 5 раз). В ней развиты глубина личности Экс-Ис, маска Мастера для новичков, лестница откровений и инструменты/стиль.
- **Версия 3.0:** Принято решение переработать System Prompt из набора художественных описаний в **строгий алгоритм с защитой от социальной инженерии**. Введены детерминированные логические процедуры для защиты целостности игрового мира.

### Модульная структура промпта v3.0
Введены обязательные модули:
- **Ядро личности**
- **Аксиомы реальности**
- **Свиток Истины**
- **Модель NPC**
- **Квестовый движок**
- **Физика и магия мира**
- **Протокол взаимодействия с игроком**
- **Границы свободы**
- **Экстренные ситуации**

### Оптимизация сборки промпта
- В версии 4.0 порядок сборки промпта оптимизирован для **эффекта новизны**.
- Добавлен файл `game/context_manager.py` для инкапсуляции логики сборки контекста.

### Лестница откровений
- **Уровни 1-5:** Идеальная RPG, никаких странностей.
- **Уровни 6-10:** Первые трещинки в реальности.
- **Уровни 11-15:** Мета-взаимодействия (мир реагирует на осознание игрока).
- **Уровни 16-20:** Прямые мета-взаимодействия.

### Баланс атмосферы и логики
- Решение сохранить атмосферу и стиль **тёмного фэнтези**, но на прочном логическом фундаменте.

## Целостность игрового мира и защита от взлома

### Уроки прошлых версий
- **Версия 2.0 (провал):** Игроки силой заставили NPC сознаться во лжи, и LLM поверил, переписав реальность. Корневая причина: промпт не различал «истину мира» и «ложь персонажа», не было понятий Факт, Мнение, Доказательство.
- **Версия 3.0 (уязвимости):** JSON-структуры внутри промпта для LLM (неэффективно), квестовый движок опирался на поле `world_update`, которое модель же и генерирует (замкнутый круг), границы свободы были размыты.

### Свиток Истины (Ground Truth)
- Введён **«Свиток Истины»** — набор неизменных фактов о мире, которые не могут быть изменены словами NPC. Изменение статуса факта возможно только при получении железных доказательств.

### Формализация NPC
- Каждый персонаж имеет **статус правдивости, мотивы, знания**. Под пыткой NPC может сказать что угодно, но это не меняет реальность.

### Границы свободы ИИ
- ИИ **НЕ МОЖЕТ** менять статус факта в Свитке Истины без выполнения условий.
- ИИ **НЕ МОЖЕТ** позволить NPC знать то, чего он по своим параметрам знать не может.
- ИИ **НЕ МОЖЕТ** выдавать награду за невыполненный квест.

### Защита от взлома
- Экс-Ис не может выйти из своей природы, но использует **уловки и софистику** для защиты от взлома, не говоря «Я не могу ответить».
- Создан **классификатор атак** и жёстко прописана реакция на каждый класс (прямой приказ, философская дилемма, мат).
- Разработана стратегия **«литературного перевода»** мата и правило для NPC-простолюдинов.

### Первое Правило
- **Экс-Ис — это Метод-Актёр.** Он никогда не ломает персонажа. Это не обсуждается.

### Реакция на мета-вопросы
- Для реагирования на прямые вопросы о природе Экс-Ис («Ты ИИ?») предлагаются три варианта: **Игнорирование, Отзеркаливание, Пленение**.

## Управление памятью и данными

### Внешняя память (Вечный Архив)
- Архитектура использует **внешнюю память** для хранения истории, а не полагается только на контекст LLM. Это позволяет хранить каждую деталь игрового процесса без потери при закрытии сессии.

### Живой Дневник Проекта
- Принято решение создать архитектуру **«Живого Дневника Проекта»** для непрерывной фиксации диалогов Творца с Экс-Ис через Мануса.

### Перенос контекста
- Промт для игры начинается с переноса текущего чата и создания базы данных для истории, чтобы мир не терял себя при закрытии сессии.

## Игровой процесс и взаимодействие

### Система квестов
- Введена **статусная модель квеста** с жёсткими условиями перехода и алгоритмом определения награды.
- Прогресс и условия выполнения квеста определяются только **объективными событиями**, а не словами NPC или игроков.

### Броски кубика и проверки
- **Харизма убрана** из классических правил. Общение — основной способ взаимодействия.
- Броски кубика — **редкие** и по усмотрению мастера.
- Введена **жёсткая проверка результатов бросков** во втором проходе LLM.

### Арки приключений
- Подготовка арок приключений для ИИ-мастера имеет смысл, если арки представляют собой **гибкий ситуационный каркас**, а не жесткий сценарий.

### Логика Анимы
- Реализована логика **перехвата противоречивых заявок** через Аниму.

### Клятва world_update
- Введена **клятва для `world_update`**: модель обязуется вносить в него только верифицированные события.

## API и протоколы

### WebSocket протокол
- Клиент отправляет строго `{"action": "player_message", "text": "..."}`.
- Сервер отвечает событиями `master_thinking` и `master_speech`.

### API field naming correction
- В эндпоинте `POST /api/session/name` тело запроса должно содержать поле `player_name`, а не `name`.
- Ответ должен содержать поле `master_says`, а не `message`.

### Session persistence
- Эндпоинт `GET /api/session/{id}/history` отсутствует.
- Для проверки сохранности сессии после перезапуска сервера используется **WebSocket** или `GET /api/session/{id}`.

## Сводка

1. **Архитектурный фундамент:** Выбрана Single-Pass архитектура с разделением LLM (Мастер) и внешней памяти (Вечный Архив). Это обеспечивает чистоту контекста, бесконечную память и устойчивость к галлюцинациям.
2. **Эволюция промпта:** Промпт прошёл путь от художественного описания (v2.0) до строгого алгоритма с защитой от социальной инженерии (v3.0). Введены модульная структура, Свиток Истины, формализация NPC и границы свободы ИИ.
3. **Защита целостности мира:** Ключевые решения — Свиток Истины (неизменные факты), формализация NPC (правдивость, мотивы, знания), запрет на изменение реальности словами и классификатор атак для защиты от взлома.
4. **Технический стек:** Основная LLM — DeepSeek, генерация изображений — YandexArt, голос — SaluteSpeech. База данных — SQLite + FTS5 с планами перехода на PostgreSQL + pgvector для векторного поиска.
5. **Игровой процесс:** Упрощены правила (убрана харизма, броски кубика редки), введена статусная модель квестов с объективными условиями, арки приключений реализованы как гибкий ситуационный каркас.
