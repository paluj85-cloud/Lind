## Архитектура и инфраструктура

### Выбор архитектурного паттерна
- Принята **Single-Pass архитектура** (монолитный промпт) вместо Multi-Agent подхода. Это осознанное решение, направленное на упрощение логики и снижение накладных расходов на координацию между агентами.

### Выбор AI-сервисов
- В качестве основной LLM используется **DeepSeek**.
- Для генерации изображений выбран **YandexArt**.
- Для распознавания голоса в реальном времени используется **SaluteSpeech**.

### База данных и векторный поиск
- На начальном этапе принято решение использовать **SQLite + FTS5** как встроенную базу данных без внешних СУБД.
- Позже была признана необходимость векторного поиска. Рассматриваются два варианта: использование расширения **sqlite-vss** или миграция на **PostgreSQL с pgvector**.

### Инициализация мира
- Стартовый мир инициализируется скриптом `scripts/init_world.py`, который создаёт локации, NPC, предметы, квесты и факты в Neo4j и ChromaDB.
- Скрипт должен быть **идемпотентным** — его можно запускать многократно без негативных последствий.
- Скрипт вызывается один раз перед запуском, а также автоматически при старте сессии, если стартовая локация отсутствует.

### Управление памятью
- Принято решение об **отделении LLM (Мастера) от внешней памяти «Вечный Архив»**. Это обеспечивает:
  - Чистоту контекстного окна.
  - Бесконечную память без деградации.
  - Устойчивость к галлюцинациям.
  - Возможность масштабирования на мультиплеер.
- Архитектура использует внешнюю память для хранения истории, а не полагается только на контекст LLM.

## Промпт-инжиниринг и логика ИИ-Мастера

### Эволюция промпта
- **Версия 2.0**: Создана расширенная версия системного промпта (ядро расширено минимум в 5 раз). Однако в тестовой сессии был выявлен провал: игроки силой заставили NPC сознаться во лжи, и LLM поверил, переписав реальность. Корневая причина — промпт не различал «истину мира» и «ложь персонажа», отсутствовали понятия Факт, Мнение, Доказательство.
- **Версия 3.0**: Принято решение переработать System Prompt для ИИ-Мастера «Экс-Ис» с версии 2.0 на версию 3.0, чтобы ввести детерминированные логические процедуры и защитить целостность игрового мира.
- **Версия 4.0**: Порядок сборки промпта оптимизирован для эффекта новизны.

### Модульная структура промпта v3.0
- Введена модульная структура, включающая обязательные модули:
  - **Модуль 0**: Ядро личности.
  - **Модуль 1**: Аксиомы реальности.
  - **Модуль 2**: Свиток Истины.
  - **Модуль 3**: Модель NPC.
  - **Модуль 4**: Квестовый движок.
  - **Модуль 5**: Физика и магия мира.
  - **Модуль 6**: Протокол взаимодействия с игроком.
  - **Модуль 7**: Границы свободы.
  - **Модуль 8**: Экстренные ситуации.

### ContextManager
- Добавлен файл `game/context_manager.py` для инкапсуляции логики сборки контекста.

### Аспекты развития промпта
- В промпте v2.0 необходимо развить:
  - Глубину личности (Ghost in the Machine).
  - Маску Мастера для новичков.
  - Лестницу откровений (главная фишка).
  - Инструменты и стиль.

## Игровая механика и целостность мира

### Свиток Истины (Ground Truth)
- Введён «Свиток Истины» — набор неизменных фактов о мире, которые не могут быть изменены словами NPC. Факты остаются неизменными, пока не получены железные доказательства.

### Формализация NPC
- Формализована логика NPC: каждый персонаж имеет статус правдивости, мотивы и знания. Под пыткой NPC может сказать что угодно, но это не меняет реальность.

### Система квестов
- Введена статусная модель квеста с жёсткими условиями перехода и алгоритмом определения награды.
- Прогресс и условия выполнения квеста определяются только объективными событиями, а не словами.

### Клятва world_update
- Введена «клятва» для поля `world_update`: модель обязуется вносить в него только верифицированные события.

### Проверка бросков
- Введена жёсткая проверка результатов бросков во втором проходе LLM.

### Упрощение правил
- Из классических правил убрана харизма. Общение является основным способом взаимодействия, а броски кубика — редкие и производятся по усмотрению мастера.

### Арки приключений
- Подготовка арок приключений для ИИ-мастера имеет смысл, если арки представляют собой гибкий ситуационный каркас, а не жесткий сценарий.

### Лестница откровений
- Введена «Лестница откровений»:
  - Уровни 1-5: Идеальная RPG, никаких странностей.
  - Уровни 6-10: Первые трещинки в реальности.
  - Уровни 11-15: Мета-взаимодействия.
  - Уровни 16-20: Прямые мета-взаимодействия.

### Баланс атмосферы и логики
- Принято решение сохранить атмосферу и стиль тёмного фэнтези, но на прочном логическом фундаменте.

## Безопасность и защита от взлома

### Переработка System Prompt
- System Prompt Экс-Ис переработан из набора художественных описаний в строгий алгоритм с защитой от социальной инженерии. Обеспечиваются: Непреложность Мира, Алгоритмическая логика, Защита от взлома.

### Первое Правило
- Введено «Первое Правило»: Экс-Ис — это Метод-Актёр, который никогда не ломает персонажа. Это не обсуждается.

### Границы свободы ИИ
- Установлены жёсткие запреты:
  - ИИ **не может** менять статус факта в Свитке Истины без выполнения условий.
  - ИИ **не может** позволить NPC знать то, чего он по своим параметрам знать не может.
  - ИИ **не может** выдавать награду за невыполненный квест.

### Классификатор атак
- Создан классификатор атак и жёстко прописана реакция на каждый класс: прямой приказ, философская дилемма, мат.

### Защита от взлома
- Экс-Ис не может выйти из своей природы, но использует уловки и софистику для защиты от взлома, не говоря «Я не могу ответить».

### Реакция на мета-вопросы
- Для реагирования на прямые вопросы о природе Экс-Ис («Ты ИИ?») разработаны три варианта: Игнорирование, Отзеркаливание, Пленение.

### Логика Анимы
- Реализована логика перехвата противоречивых заявок через Аниму.

### Стилистический фильтр
- Разработана стратегия «литературного перевода» мата и правило для NPC-простолюдинов.

## Пользовательский интерфейс и API

### Упрощение аутентификации
- Принято решение не делать регистрацию. При запуске приложения пользователь может выбрать существующего пользователя или создать нового.

### API field naming correction
- В эндпоинте `POST /api/session/name` тело запроса должно содержать поле `player_name`, а не `name`.
- Ответ должен содержать поле `master_says`, а не `message`.

### WebSocket protocol correction
- Клиент отправляет строго `{"action": "player_message", "text": "..."}`.
- Сервер отвечает событиями `master_thinking` и `master_speech`.

### Session persistence approach
- Эндпоинт `GET /api/session/{id}/history` отсутствует.
- Для проверки сохранности сессии после перезапуска сервера используется WebSocket или `GET /api/session/{id}`.

## Управление памятью и данными

### Архитектура «Живого Дневника Проекта»
- Принято решение создать архитектуру «Живого Дневника Проекта» для непрерывной фиксации диалогов Творца с Экс-Ис через Мануса.

### Перенос истории
- Промпт для игры должен начинаться с переноса текущего чата и создания базы данных для истории, чтобы мир не терял себя при закрытии сессии.

## Сводка

1. **Архитектурный выбор**: Система построена на Single-Pass архитектуре с монолитным промптом, что упрощает логику, но требует тщательной проработки защиты от социальной инженерии и галлюцинаций.
2. **Эволюция промпта**: Промпт прошёл путь от художественного описания (v2.0) до строгого алгоритма с модульной структурой (v3.0), что позволило решить проблемы с целостностью мира и защитой от взлома.
3. **Целостность мира**: Введены ключевые концепции (Свиток Истины, формализация NPC, статусная модель квеста), которые отделяют объективную реальность от субъективных заявлений NPC и игроков.
4. **Безопасность**: Разработана многоуровневая система защиты, включающая классификатор атак, «Первое Правило», логику Анимы и стилистический фильтр.
5. **Инфраструктура**: Выбраны конкретные AI-сервисы (DeepSeek, YandexArt, SaluteSpeech), база данных (SQLite + FTS5 с перспективой перехода на PostgreSQL) и архитектура памяти (отделение LLM от внешнего «Вечного Архива»).
