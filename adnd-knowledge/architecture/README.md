## Общая архитектура

- Архитектура Экс-Ис v5.0 использует Single-Pass архитектуру с Entity System, Context Request, World Pulse, Dynamic Relationships, AI-driven Gossip, Procedural Quest Generation & Milestones, Emotional NPC Model, World Time, Significance System, Quest Classification, Subjective Completion & Causal Stability.
- Исходная архитектура включает чат-интерфейс (Frontend), API-сервер (FastAPI), менеджер памяти (контекстный менеджер, индексатор файлов, индексатор БД, экстрактор фактов), индексы PostgreSQL с pgvector (позже пересмотрено на SQLite) и инструменты для LLM: read_file, run_query, list_directory, search_code.
- Закладывается разделение личного знания игрока (session_private) и мирового канона (world_shared).
- Архитектура включает Аниму — первое дитя, Искру, которая живёт на границе между тюрьмой Экс-Ис и сознанием игрока, выступая мостом и советником.
- Реализована логика перехвата противоречивых заявок через Аниму.

## Технологический стек

- **Язык:** Python 3.11+ (строго).
- **Бэкенд-фреймворк:** FastAPI + Uvicorn (асинхронный).
- **Базы данных:**
    - SQLite с полнотекстовым поиском FTS5 (с возможностью перехода на PostgreSQL с pgvector).
    - MySQL 8, InnoDB, префикс таблиц `arc_`.
    - Neo4j для графовой БД.
    - ChromaDB для векторного поиска.
- **ORM и миграции:** SQLAlchemy 2.0 (asyncio), aiomysql (для MySQL 8), Alembic.
- **Валидация:** Pydantic v2.
- **Конфигурация:** pydantic-settings (.env), загрузка через python-dotenv.
- **Логирование:** Структурированное логирование (logging/structlog).
- **Фоновые задачи:** BackgroundTasks + asyncio.create_task с ограничением параллелизма.
- **Фронтенд:** Vanilla HTML/CSS/JS.
- **Тестирование:** pytest + httpx + pytest-asyncio.
- **Деплой:** VPS с Nginx и systemd.

## Бэкенд

- Бэкенд приложения написан на Python с использованием FastAPI и поддерживает REST и WebSocket эндпоинты.
- Бэкенд реализуется на FastAPI (Python 3.11+), работает асинхронно. Длительная обработка ведётся в фоне с обновлением состояния в БД; фронтенд опрашивает прогресс.
- Бэкенд реализован на FastAPI с WebSocket для чата.
- Приложение инициализирует Neo4jRepository, ChromaRepository, ConnectionManager и SessionManager в lifespan.

## Базы данных и хранилища

### SQLite / MySQL

- База данных приложения — SQLite с полнотекстовым поиском FTS5.
- Приложение использует SQLite базу данных с таблицами `chronicles` и `chronicles_fts` (FTS5).
- База данных — MySQL.
- База данных MySQL 8, InnoDB, префикс таблиц `arc_`. Содержит справочники (`arc_tag_category`, `arc_tag`, `arc_mask`, `arc_sensory_type`, `arc_detail_type`, `arc_location_archetype`, `arc_tone`), основные таблицы атомов (`arc_conflict`, `arc_twist`, `arc_dilemma`, `arc_relationship`, `arc_location`, `arc_sensory_hook`, `arc_npc_detail`), таблицы для процесса анализа (`arc_prompt`, `arc_book`, `arc_fragment`, `arc_draft_*` для каждого типа атомов, `arc_processing_state`).

### Neo4j и ChromaDB

- Проект использует FastAPI с WebSocket и REST API, Neo4j для графовой БД, ChromaDB для векторного поиска, и DeepSeek LLM для генерации.
- Стартовый мир инициализируется скриптом `scripts/init_world.py`, который создаёт локации, NPC, предметы, квесты и факты в Neo4j и ChromaDB.

### Human-память

- Human-память включает хроники мира в формате Markdown и мета-информацию в SQLite/PostgreSQL.
- Хроники Мира: ИИ должен параллельно генерировать человекочитаемый лог произошедшего в формате художественного повествования (Markdown).

## Внешняя память: «Вечный Архив»

- Система включает Вечный Архив — внешнюю память, которая хранит историю и подаёт только релевантный контекст.
- Архитектура внешней памяти «Вечный Архив» включает базу данных (предпочтительно PostgreSQL + pgvector, допустим SQLite + sqlite-vss), контекстный менеджер, службу канонизации, агент валидации и обработчик Анимы.
- Архитектура «Вечного Архива» адаптирована для Cline CLI через MCP-сервер, предоставляющий инструменты: `start_session`, `get_relevant_context`, `add_project_fact`, `add_seed`, `summarize_session` и т.д.
- Адаптированная модель данных для Cline: Явленный Свиток → Проектные факты, Тайный Свиток → Контекст сессии, Семена → Технический долг и идеи, Нити судьбы → Активные ветки разработки, Канонизация → Суммаризация сессии.
- Добавлен файл `game/context_manager.py` для инкапсуляции логики сборки контекста.
- Предлагается архитектура памяти для хранения непреложных фактов: рассматриваются варианты векторная БД, реляционная БД или гибрид.

## ИИ-агенты и интеграции

- Приложение интегрировано с DeepSeek API.
- Основная LLM — DeepSeek, генерация картинок — YandexArt, голос — SaluteSpeech (распознавание и синтез).
- Для точечных изменений кода используется FIM (Fill-In-the-Middle) эндпоинт DeepSeek API.
- ИИ-агент Манус (Cline) будет проводить автоматизированные тесты.

## Системный промпт (v5.0)

- System Prompt v5.0 должен включать процедуру резюмирования и канонизации событий для экономии контекста.
- System Prompt v5.0 должен быть масштабируемым на мультиплеер и долгие кампании.
- System Prompt v5.0 должен иметь идеальную структуру для парсинга ответов (например, с использованием тегов).
- System Prompt v5.0 должен включать продвинутую защиту от мета-игры и социальной инженерии игроков.
- System Prompt v5.0 должен содержать протокол поведения в экстренных ситуациях (логические дыры, убийство ключевых NPC, застревание игроков).
- Результат v5.0 — это не только текст промпта, но и Python-класс `SystemPromptBuilder`, собирающий его из модулей с учётом динамических параметров (`reveal_level`, `player_name`, `truth_scroll`, `quest_log`, `character_sheets`, `summary_context`, `world_lore`).

## Администрирование и развёртывание

- Есть страница администратора для настройки промптов, логики игры и связей между ИИ, а также для подключения/смены разных ИИ.
- Предполагается, что сервер управляется через systemd.
- В корне проекта создаётся файл `ORIGIN.md`, содержащий весь диалог Творца с Экс-Ис; при инициализации репозитория Манус копирует чат в `ORIGIN.md` и добавляет ссылку в `.cline/memory/productContext.md`.
- `activeContext.md` должен содержать краткий перечень последних обсуждений и текущих приоритетов (максимум 15-20 строк) и обновляться после каждого значимого диалога.
- `progress.md` будет содержать раздел 'Концептуальные решения' для записи важных договорённостей, не связанных с кодом напрямую.

## Сводка

1. **Модульная архитектура:** Проект построен на асинхронном бэкенде FastAPI с разделением на чёткие компоненты: базы данных (SQLite/MySQL, Neo4j, ChromaDB), внешняя память («Вечный Архив»), ИИ-агенты и система промптов.
2. **Гибридное хранилище:** Используется комбинация реляционных (SQLite/MySQL), графовых (Neo4j) и векторных (ChromaDB) баз данных для эффективного хранения и поиска различных типов информации.
3. **Внешняя память:** Ключевым элементом является «Вечный Архив» — система, обеспечивающая долговременное хранение контекста и релевантную подачу информации, адаптированная для работы через MCP-сервер с Cline.
4. **Интеграция с DeepSeek:** Основная логика генерации и взаимодействия строится на DeepSeek API, включая FIM для точечных изменений кода.
5. **Масштабируемость и управляемость:** Архитектура закладывает разделение личного и мирового знания, поддерживает Single-Pass обработку, включает административную панель и строгие правила для системного промпта (v5.0), что обеспечивает гибкость и контроль над игровым процессом.
