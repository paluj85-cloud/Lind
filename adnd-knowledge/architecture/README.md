## Технологический стек

- **Язык**: Python 3.11+.
- **Бэкенд-фреймворк**: FastAPI + Uvicorn (асинхронный).
- **База данных**: SQLite с полнотекстовым поиском FTS5 (с возможностью перехода на PostgreSQL с pgvector).
- **Графовая база данных**: Neo4j.
- **Векторное хранилище**: ChromaDB.
- **ORM**: SQLAlchemy 2.0 (asyncio).
- **Драйвер для MySQL**: aiomysql (используется в альтернативной схеме).
- **Валидация данных**: Pydantic v2.
- **Миграции**: Alembic.
- **Фронтенд**: Vanilla HTML/CSS/JS.
- **Тестирование**: pytest + httpx + pytest-asyncio.
- **Деплой**: VPS с Nginx и systemd.
- **Логирование**: Структурированное логирование (logging/structlog).
- **Конфигурация**: pydantic-settings (.env).

## Бэкенд

- Бэкенд реализован на FastAPI (Python 3.11+), работает асинхронно.
- Поддерживает REST и WebSocket эндпоинты.
- Длительная обработка выполняется в фоне с обновлением состояния в базе данных; фронтенд опрашивает прогресс.
- Для чата используется WebSocket.

## База данных и хранилища

### Основная база данных
- Приложение использует SQLite базу данных с таблицами `chronicles` и `chronicles_fts` (FTS5).
- Альтернативная схема: MySQL 8, InnoDB, префикс таблиц `arc_`. Содержит справочники (`arc_tag_category`, `arc_tag`, `arc_mask`, `arc_sensory_type`, `arc_detail_type`, `arc_location_archetype`, `arc_tone`), основные таблицы атомов (`arc_conflict`, `arc_twist`, `arc_dilemma`, `arc_relationship`, `arc_location`, `arc_sensory_hook`, `arc_npc_detail`), таблицы для процесса анализа (`arc_prompt`, `arc_book`, `arc_fragment`, `arc_draft_*` для каждого типа атомов, `arc_processing_state`).

### Графовая база данных
- Используется Neo4j для хранения и обработки связей между сущностями игрового мира.

### Векторное хранилище
- Используется ChromaDB для семантического поиска и хранения векторных представлений.

## Внешняя память «Вечный Архив»

- Система включает «Вечный Архив» — внешнюю память, которая хранит историю и подаёт только релевантный контекст.
- Компоненты «Вечного Архива»:
  - База данных (предпочтительно PostgreSQL + pgvector, допустим SQLite + sqlite-vss). Таблицы: `sessions`, `project_facts`, `session_context`, `seeds`, `threads_of_fate`.
  - Контекстный менеджер.
  - Служба канонизации.
  - Агент валидации.
  - Обработчик Анимы.
- Адаптированная модель данных для Cline:
  - Явленный Свиток → Проектные факты (Project Facts).
  - Тайный Свиток → Контекст сессии (Session Context).
  - Семена → Технический долг и идеи (Seeds).
  - Нити судьбы → Активные ветки разработки (Threads of Fate).
  - Канонизация → Суммаризация сессии (Summarize Session).
- Архитектура «Вечного Архива» адаптирована для Cline CLI через MCP-сервер, предоставляющий инструменты: `start_session`, `get_relevant_context`, `add_project_fact`, `add_seed`, `summarize_session` и другие.
- Закладывается разделение личного знания игрока (`session_private`) и мирового канона (`world_shared`).
- Human-память включает хроники мира в формате Markdown и мета-информацию в SQLite/PostgreSQL.

## Искусственный интеллект и интеграции

- Основная LLM — DeepSeek.
- Приложение интегрировано с DeepSeek API.
- Для точечных изменений кода используется FIM (Fill-In-the-Middle) эндпоинт DeepSeek API.
- AI для генерации картинок — YandexArt.
- Голосовое взаимодействие: SaluteSpeech (распознавание и синтез).
- ИИ-агент Манус (Cline) будет проводить автоматизированные тесты.

## Архитектура системы v5.0

- Архитектура Экс-Ис v5.0 использует Single-Pass архитектуру с:
  - Entity System.
  - Context Request.
  - World Pulse.
  - Dynamic Relationships.
  - AI-driven Gossip.
  - Procedural Quest Generation & Milestones.
  - Emotional NPC Model.
  - World Time.
  - Significance System.
  - Quest Classification.
  - Subjective Completion & Causal Stability.
- System Prompt v5.0 должен:
  - Включать процедуру резюмирования и канонизации событий для экономии контекста.
  - Быть масштабируемым на мультиплеер и долгие кампании.
  - Иметь идеальную структуру для парсинга ответов (например, с использованием тегов).
  - Включать продвинутую защиту от мета-игры и социальной инженерии игроков.
  - Содержать протокол поведения в экстренных ситуациях (логические дыры, убийство ключевых NPC, застревание игроков).
- Результат v5.0 — это не только текст промпта, но и Python-класс `SystemPromptBuilder`, собирающий его из модулей с учётом динамических параметров (`reveal_level`, `player_name`, `truth_scroll`, `quest_log`, `character_sheets`, `summary_context`, `world_lore`).

## Структура проекта

- Структура проекта включает директории: `models`, `db`, `api`, `game`, `game/prompts`, `scripts`, `static`, `tests`.
- В корне проекта создаётся файл `ORIGIN.md`, содержащий весь диалог Творца с Экс-Ис; при инициализации репозитория Манус копирует чат в `ORIGIN.md` и добавляет ссылку в `.cline/memory/productContext.md`.
- `activeContext.md` должен содержать краткий перечень последних обсуждений и текущих приоритетов (максимум 15-20 строк) и обновляться после каждого значимого диалога.
- `progress.md` будет содержать раздел «Концептуальные решения» для записи важных договорённостей, не связанных с кодом напрямую.

## Инициализация и конфигурация

- Приложение инициализирует `Neo4jRepository`, `ChromaRepository`, `ConnectionManager` и `SessionManager` в `lifespan`.
- Конфигурация загружается из `.env` через `python-dotenv`.
- Стартовый мир инициализируется скриптом `scripts/init_world.py`, который создаёт локации, NPC, предметы, квесты и факты в Neo4j и ChromaDB.

## Администрирование и развёртывание

- Есть страница администратора для настройки промптов, логики игры и связей между ИИ, а также для подключения/смены разных ИИ.
- Предполагается, что сервер управляется через systemd.

## Сводка

1. **Гибридная архитектура хранения**: Проект использует комбинацию SQLite (с FTS5), Neo4j (графовая БД) и ChromaDB (векторное хранилище) для обеспечения эффективного хранения и поиска данных различной природы.
2. **Асинхронный бэкенд на FastAPI**: Бэкенд построен на FastAPI с поддержкой REST и WebSocket, что обеспечивает высокую производительность и возможность работы в реальном времени.
3. **Интеграция с DeepSeek и другими AI-сервисами**: Основная LLM — DeepSeek, также используются YandexArt (генерация изображений) и SaluteSpeech (голосовое взаимодействие).
4. **Внешняя память «Вечный Архив»**: Реализована система внешней памяти, которая хранит историю и подаёт релевантный контекст, с адаптацией для Cline через MCP-сервер.
5. **Модульная архитектура System Prompt v5.0**: System Prompt собирается из модулей с помощью Python-класса `SystemPromptBuilder`, что обеспечивает гибкость и масштабируемость для мультиплеера и долгих кампаний.
