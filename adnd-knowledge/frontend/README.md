# Фронтенд

Фронтенд проекта Lind (ADND) представляет собой статическое веб-приложение на HTML/CSS/JS, которое обеспечивает интерфейс для взаимодействия игроков с AI-мастером игры.

## Архитектура и структура

### Состав фронтенда
- Фронтенд состоит из статических файлов HTML/CSS/JS.
- Клиентская часть — статика для браузера, расположенная в папке `client`.
- Фронтенд представлен статическими файлами: `index.html`, `style.css`, `chat.js`, расположенными в `backend/app/static/`.
- Фронтенд — статический HTML/JS, отображает приветствие и запрос имени.

## Интерфейс пользователя

### Макет экрана
- Веб-интерфейс адаптирован для ПК и смартфона.
- В верхней части экрана отображается картинка, генерируемая на основе сообщений в чате.
- Под картинкой расположен сам чат, ниже — поле ввода сообщения.
- HP, MP и деньги отображаются слева в верхнем углу.

### Элементы интерфейса
- Иконка персонажа в правом верхнем углу открывает лист персонажа.
- Клиентский интерфейс включает:
  - Общий чат: отображает нарратив мастера и заявки игроков.
  - Панель заявки: текстовое поле и выбор типа (действие/реплика).
  - Кнопка «Конец хода».
  - Окно броска.
  - Личная вкладка Анимы.
  - Панель игрока: имя, HP%, MP%, инвентарь.
  - Служебные уведомления.

## Функциональность чата

### Минимальный UX чата
- Минимальный UX чата включает текстовое поле, историю сообщений и индикатор «Мастер думает».
- Приложение имеет индикатор «Мастер размышляет», который должен работать.

## Управление сессией и пользователями

### Сохранение сессии
- История диалога должна сохраняться и после перезагрузки страницы сессия не теряется.

### Выбор пользователя
- При запуске приложения пользователь может выбрать существующего или создать нового пользователя.
- Первый вход игрока должен быть описан в ТЗ.

## Экран приветствия

### Текст первого экрана
- Первый экран игры содержит текст из 2–3 предложений (не более 50 слов), который даёт ощущение загадки и трепета, объясняет суть игры и упоминает Аниму, завершаясь намёком-приглашением.
- Текст первого экрана не содержит прямых команд вроде «Нажми начать» или «Входи», а завершается намёком-приглашением.
- Текст первого экрана избегает слова «портал».

## Сводка

1. Фронтенд реализован как статическое веб-приложение (HTML/CSS/JS), расположенное в `backend/app/static/`.
2. Интерфейс включает чат с историей, поле ввода, индикатор «Мастер думает», а также панели HP/MP/денег и лист персонажа.
3. Сессия пользователя сохраняется между перезагрузками страницы, поддерживается выбор существующего или создание нового пользователя.
4. Экран приветствия содержит загадочный текст из 2–3 предложений с приглашением, без прямых команд и слова «портал».
5. Интерфейс адаптирован для ПК и мобильных устройств, включает генерацию изображений на основе сообщений чата.
