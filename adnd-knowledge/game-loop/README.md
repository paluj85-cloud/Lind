# Игровой цикл

Игровой цикл Lind (ADND) — это последовательность этапов от начала сессии до её завершения, включающая подготовку мира, вождение, обработку действий игроков, броски кубов и обновление игрового состояния.

## Общая структура игрового цикла

- Игровой цикл включает пять основных этапов: подготовку мира, старт игры, вождение, независимую жизнь мира и финал.
- На этапе подготовки мира ИИ создаёт Книгу Мира и Книгу Истории.
- На старте игры ИИ генерирует Книгу Завязки.
- Вождение осуществляется через фабрику арок, которая реагирует на действия игроков.
- Мир живёт независимо: NPC и таймеры действуют без прямого участия игроков.
- Финал завершается генерацией романа по итогам сессии.

## Начало игры

- После запуска игры мастер представляется и рассказывает основные правила и вводный сеттинг игры.
- Мастер просит игроков представиться и рассказать о желаемых персонажах.

## Порядок хода и сбор заявок

- При старте или после завершения предыдущего хода сервер извлекает из Neo4j всех активных игроков, перемешивает их случайным образом и рассылает порядок хода.
- Каждый игрок через клиент отправляет заявку с полями `player`, `action_type` (`say` или `do`) и `content`.
- Игроки могут отправлять заявки (действия) или задавать уточняющие вопросы мастеру.
- Пустая заявка может означать пропуск хода или поддержание предыдущего действия.
- Если заявка пустая, сервер проверяет предыдущую заявку из истории хода: если прошлое действие было `do` с пометкой «поддержание» и не было прерывания, то текущая заявка автоматически заполняется как `do` с content «продолжает поддерживать ...». В противном случае игрок ничего не делает.

## Обработка заявок и контекста

- После получения всех заявок сервер собирает контекст: последние N реплик и результаты FTS5-поиска.
- Мастер может перед ответом задать уточняющий вопрос игроку, чтобы лучше понять его намерения.

## Формирование промпта и вызов LLM

- Первый вызов Экс-Ис формирует промпт со следующими полями:
  - СЕТТИНГ, ПАРТИЯ, УРОВЕНЬ ОТКРОВЕНИЯ, МИРОВОЕ ВРЕМЯ — константы сессии.
  - ПОРЯДОК ДЕЙСТВИЙ, ЗАЯВКИ ИГРОКОВ, АВАТАРЫ, КОНТЕКСТ ПАМЯТИ, МИРОВОЙ ЛОР, ТАЙНЫЙ СВИТОК, ЯВЛЕННЫЙ СВИТОК, НИТИ СУДЬБЫ, СЕМЕНА БУДУЩЕГО, ТЕКУЩИЕ СУЩНОСТИ.
  - РЕЗУЛЬТАТЫ БРОСКОВ — отсутствует в первом вызове.
- Полный промпт отправляется в DeepSeek.

## Обработка ответа LLM

- Экс-Ис общается через JSON с полями `narrator`, `world_update` и `dice_request`.
- Сервер получает ответ модели и парсит его.
- Если в ответе есть теги `<dice_request>`, сервер отправляет запрос броска игроку.
- Если тегов нет, ответ считается финальным, и сервер переходит к обновлению мира.

## Обработка бросков кубов

- Сервер парсит ответ LLM на наличие тегов `<dice_request>`.
- Если теги обнаружены, сервер отправляет запрос броска игроку, получает результат, сравнивает с DC (сложностью) и определяет нарративный статус.
- После обработки броска результат возвращается в цикл для формирования финального ответа.

## Обновление мира

- Поле `world_update` заполняется только для значимых событий.
- Допустимые типы событий: `event`, `entity_change`, `location_change`, `item_change`.
- Для каждого события указывается `narrative_summary` длиной до 100 символов.
- Если в ответе LLM есть `world_update`, он асинхронно сохраняется в БД.
- При достижении порога несжатых событий запускается фоновая саммаризация.

## Протокол WebSocket

- Клиент отправляет сообщение через WebSocket.
- Сервер отправляет `master_thinking` со статусом `started`.
- Сервер собирает контекст, формирует промпт и вызывает DeepSeek `generate()`.
- Сервер парсит `LLMResponse`.
- Сервер отправляет `master_speech` с полем `narrator`.
- Если есть `world_update`, он асинхронно сохраняется в БД и проверяется порог саммаризации.
- Сервер отправляет `master_thinking` со статусом `stopped`.

## Сводка

1. Игровой цикл состоит из пяти этапов: подготовка мира, старт, вождение, жизнь мира и финал, каждый из которых автоматизирован ИИ.
2. Каждый ход начинается с рассылки порядка хода, сбора заявок игроков и формирования контекста для LLM.
3. Пустые заявки автоматически интерпретируются как поддержание предыдущего действия или пропуск хода.
4. Ответ LLM может содержать запрос на бросок кубов, который обрабатывается сервером с последующим обновлением нарратива.
5. Обновление мира происходит только для значимых событий с кратким описанием, а при накоплении несжатых событий запускается фоновая саммаризация.
