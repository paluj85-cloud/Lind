# AI-оптимизированная база знаний ADND — Предложенная структура

> Сохранено 2026-05-15 для продолжения после перезапуска Cline.

---

## Двухуровневая AI-читаемая структура

### Уровень 1: Точка входа (один файл)

```
adnd-knowledge/
├── README.md              ← «Прочитай меня первым»
│                            (для Cline: что это, карта, стратегия чтения)
```

### Уровень 2: Тематические файлы (плоская иерархия, макс. 2 уровня)

```
adnd-knowledge/
├── README.md                          ← Точка входа + карта + стратегия чтения
│
├── 01-vision-and-philosophy.md        ← Идея, философия, «что мы строим»
│   (Откуда взялся Экс-Ис, Древо Миров, Бог-Узник, Завет Молчания)
│
├── 02-game-design.md                  ← Игровые механики
│   (d20, 6 атрибутов, магия=химия, Книги Мира/Истории/Завязки,
│    Явленный и Тайный Свитки, Стражи, beyond-механика)
│
├── 03-system-prompts/                 ← Промты (отдельная папка — самые ценные)
│   ├── README.md                      ← Объяснение архитектуры промтов
│   ├── exis-master.md                 ← Системный промт Экс-Ис (Бог-Узник)
│   ├── anima-assistant.md             ← Системный промт Анимы (искра души)
│   ├── world-creation.md              ← Промт сотворения мира
│   └── greeting.md                    ← Промт приветствия игрока
│
├── 04-architecture.md                 ← Техническая архитектура
│   (FastAPI, Neo4j, ChromaDB, SQLite FTS5, WebSocket, модули C3)
│
├── 05-game-loop.md                    ← Игровой цикл (подробно)
│   (Порядок хода, сбор заявок, dice_request, два прохода LLM,
│    голосование, beyond/возврат, world_pulse)
│
├── 06-api-and-websocket.md            ← API спецификация
│   (REST эндпоинты, WebSocket сообщения, форматы JSON)
│
├── 07-frontend.md                     ← Фронтенд
│   (SPA, общий чат, панель Анимы, панель игрока, голосование)
│
├── 08-tech-specs/                     ← Технические задания по версиям
│   ├── v0.1-mvp.md                    ← ТЗ v0.1 (одиночная игра)
│   └── v0.2-multiplayer.md            ← ТЗ v0.2 (мультиплеер, миры, персонажи)
│
├── 09-decisions.md                    ← ADR: ключевые решения и «почему»
│   (Почему Neo4j, почему два прохода LLM, почему ChromaDB,
│    почему DC скрыто, почему beyond а не disconnect)
│
├── 10-implementation-status.md        ← Статус: что сделано, что в работе
│
└── 11-source-map.md                   ← Карта исходных чатов
    (Где что обсуждалось: chat_id → тема, с перекрёстными ссылками)
```

## Обоснование практиками

| Практика | Как применено |
|----------|--------------|
| **Diátaxis** | Файлы промтов (Reference), архитектура (Explanation), игровой цикл (How-to), видение (Tutorial) |
| **C4 Model** | 04-architecture.md: Контекст → Контейнеры (FastAPI, Neo4j, ChromaDB) → Компоненты (17 модулей) |
| **ADR** | 09-decisions.md: каждое решение с контекстом, альтернативами и обоснованием |
| **Readme-Driven** | README.md — точка входа с картой и стратегией чтения для ИИ |
| **AI-оптимизация** | Плоская структура, самодостаточные файлы, перекрёстные ссылки, маркеры типа |

## Контекст: что дальше

Эта структура предложена Cline в другом чате. Пользователь хочет перезапустить Cline и продолжить с этого места. Следующий шаг — утвердить структуру и приступить к наполнению файлов контентом из исходных чатов ADND.
