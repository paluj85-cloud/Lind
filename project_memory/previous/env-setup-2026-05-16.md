# Prompt: Настройка переменных окружения (DeepSeek + GitHub)

---

<role>
Ты — DevOps-инженер проекта Lind (ADND). Ты отвечаешь за конфигурацию systemd-сервисов и переменных окружения на VPS (Ubuntu, пользователь exis, корень проекта /opt/lind).
</role>

<task>
Создать файл /opt/lind/.env с ключами DeepSeek и GitHub, перевести systemd-сервис lind-ttyd на EnvironmentFile, обновить cline_launcher.sh, добавить .env в .gitignore, и перезапустить сервис.
</task>

<context>
## Проблема
- systemd-сервис `lind-ttyd.service` содержит хардкод `CLINE_API_KEY=SET_AFTER_FIRST_RUN` (заглушка вместо реального ключа)
- Переменные `DEEPSEEK_API_KEY` и `GITHUB_TOKEN` не установлены в окружении
- Backend ожидает `DEEPSEEK_API_KEY` через `os.getenv()`
- Git-скриптам нужен `GITHUB_TOKEN`

## Источники ключей
- `.cline_api_key` → содержит ключ DeepSeek (REDACTED)
- `.git-credentials` → содержит GitHub PAT (REDACTED)

## Файлы, которые нужно изменить
1. **Создать:** `/opt/lind/.env` (chmod 600)
2. **Создать:** `/opt/lind/.gitignore` (добавить `.env`)
3. **Изменить:** `/opt/lind/systemd/lind-ttyd.service`
   - Текущее: `Environment="CLINE_API_KEY=SET_AFTER_FIRST_RUN"` и `Environment=PATH=...` и `Environment=HOME=...`
   - Заменить на: `EnvironmentFile=/opt/lind/.env`
4. **Изменить:** `/opt/lind/cline_launcher.sh`
   - Добавить source `.env` как фоллбэк для интерактивных сессий (после проверки `.cline_api_key`, до `cd /opt/lind`)
5. **НЕ трогать:** `.cline/config.json`, `.git-credentials`, `.cline_api_key` — оставить как есть

## Команды после изменений
```bash
sudo systemctl daemon-reload
sudo systemctl restart lind-ttyd
env | grep -E "^(CLINE|DEEPSEEK|GITHUB)"  # проверка
```
</context>

<format>
Результат — изменённые файлы в рабочей директории /opt/lind:
1. `/opt/lind/.env` — новый файл с тремя переменными
2. `/opt/lind/.gitignore` — новый или дополненный файл
3. `/opt/lind/systemd/lind-ttyd.service` — изменён (EnvironmentFile вместо Environment)
4. `/opt/lind/cline_launcher.sh` — изменён (добавлен source .env)

Плюс выполнение команд:
- `sudo systemctl daemon-reload`
- `sudo systemctl restart lind-ttyd`
</format>

<constraints>
- НЕ менять `.cline/config.json` — Cline сам его читает
- НЕ менять `.git-credentials` — Git сам его читает
- НЕ удалять `.cline_api_key` (оставить, не мешает)
- Файл `.env` должен иметь права 600 (владелец exis)
- `.env` ДОЛЖЕН быть в `.gitignore` — никогда не коммитить секреты
- Проверить финальный результат через `env | grep`
- После systemctl restart проверить статус сервиса: `sudo systemctl status lind-ttyd`
</constraints>
