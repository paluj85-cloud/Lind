# План: Восстановление сессии при перезагрузке страницы

> **Дата**: 2026-05-16
> **Трек**: Инженер
> **Проблема**: Повтор приветствия + потеря истории при перезагрузке / реконнекте

---

## Текущее поведение

1. При каждом WebSocket-соединении сервер создаёт **новый** `session_id` (uuid4)
2. Фронтенд шлёт `player_join` → сервер запускает приветствие (2 шага)
3. При перезагрузке страницы — всё по новой, история теряется

---

## Варианты решения

### Вариант A: Восстановление сессии по session_id в localStorage

**Механизм:**
- Фронтенд сохраняет `session_id` в `localStorage`
- При новом WebSocket-соединении шлёт `player_join` с `session_id` из localStorage (если есть)
- Сервер: если сессия существует и активна → не создаёт новую, загружает историю, шлёт backlog, **не** запускает приветствие
- Сервер: если сессии нет (истекла, удалена) → создаёт новую

**Что изменить:**
1. **app.js**: добавить `localStorage` для `sessionId`, при коннекте передавать в `player_join`
2. **main.py**: в `player_join` проверять `session_id`, если передан — искать существующую сессию
3. **GameLoop**: добавить метод `restore_session()` — загружает историю и шлёт backlog
4. **Протокол**: новое сообщение `master_speech` с полем `backlog: [...]` для истории

**Плюсы:**
- Игрок возвращается в ту же сессию, видит полную историю
- Приветствие не повторяется
- Минимально инвазивно для существующей архитектуры

**Минусы:**
- Если localStorage очищен — сессия теряется (но это нормально)
- Нужно добавить backlog-сообщение в протокол

---

### Вариант B: Единая сессия по имени игрока

**Механизм:**
- Сервер ищет активную сессию по `player_name`
- Если находит — переиспользует, не запускает приветствие
- Если нет — создаёт новую

**Что изменить:**
1. **main.py**: в `player_join` искать сессию по `player_name`, переиспользовать
2. **GameLoop**: метод `restore_session()`

**Плюсы:**
- Очень просто (1-2 строки в main.py)
- Не требует изменений фронтенда

**Минусы:**
- Не работает если два игрока с одинаковым именем
- Не решает проблему с потерей истории при дисконнекте (не шлёт backlog)

---

### Вариант C: Полноценный reconnect с backlog

**Механизм:**
- Как Вариант A, но с полноценным протоколом reconnect
- Сервер не только восстанавливает сессию, но и шлёт **все** сообщения истории в виде backlog
- Фронтенд отрисовывает историю до того как игрок начнёт ввод

**Что изменить:**
1. **app.js**: `localStorage` + рендер backlog при получении
2. **main.py**: reconnect-логика
3. **GameLoop**: `restore_session()` с полным backlog
4. **Протокол**: `session_restored { session_id, backlog: [...], world_lore: [...] }`

**Плюсы:**
- Полное решение — игрок видит всю историю, все факты мира
- Правильно архитектурно — разделены новая сессия и восстановление

**Минусы:**
- Больше изменений (~60-80 строк кода)
- Нужно тестировать оба пути: новая сессия и восстановление

---

## Рекомендация

**Вариант A** — баланс между простотой и полнотой решения. Минимальные изменения, решает обе проблемы (повтор приветствия + история).
