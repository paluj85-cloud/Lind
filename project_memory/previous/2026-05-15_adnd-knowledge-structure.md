# План: Создание структуры базы знаний ADND

Дата: 2026-05-15
Фаза: Планирование
Статус: Утверждён (2026-05-15 16:08)

---

## Задача

Создать структуру `adnd-knowledge/` в `/opt/lind/` — AI-читаемую базу знаний по проекту ADND (настольная ролевая игра с AI-мастером).

---

## Варианты подхода

### Вариант A: «Пустая структура» (рекомендую)
Создать все папки и README.md с заглушками («TBD: описание появится после анализа чатов»). Наполнение — отдельным шагом.

**Плюсы**:
- Структура готова сразу, видна вся карта
- Можно наполнять постепенно, файл за файлом
- Git-friendly: каждый наполненный файл — отдельный коммит

**Минусы**:
- Пустые файлы выглядят незавершённо
- Нужен второй шаг (наполнение)

### Вариант B: «Структура + наполнение из чатов за один проход»
Сразу создать структуру и наполнить каждый README.md контентом из соответствующих чатов `chat_deepseek/`.

**Плюсы**:
- Всё и сразу — готовая база знаний после выполнения

**Минусы**:
- Огромный объём работы для одного чата Cline (115 чатов, 8.1 MB)
- Риск потери контекста из-за лимита токенов
- Высокая вероятность ошибок/пропусков при однопроходной обработке

### Вариант C: «Гибрид» — структура + приоритетное наполнение
Создать структуру. Затем наполнить в первую очередь **критичные файлы** (vision, game-design, system-prompts). Остальное — позже.

**Плюсы**:
- Максимальная ценность сразу (промты + механики + философия)
- Остальное можно доделать когда угодно
- Реалистично для одного чата

**Минусы**:
- Нужно приоритезировать: что считать «критичным»
- Часть базы останется незаполненной временно

---

## Рекомендация

**Вариант A** — сначала создать пустую структуру (это быстро и даёт каркас), а наполнение планировать отдельно по приоритетам.

Причина: 115 чатов невозможно качественно обработать за один проход Cline. Нужно разбить на сессии.

---

## Шаги (для варианта A)

### Шаг 1: Создать корневой README.md
- `/opt/lind/adnd-knowledge/README.md` — точка входа: что это за база, карта папок, стратегия чтения для AI

### Шаг 2: Создать папки и README.md в каждой
```
adnd-knowledge/
├── README.md
├── vision/README.md
├── game-design/README.md
├── system-prompts/README.md
│   ├── exis-master.md
│   ├── anima-assistant.md
│   ├── world-creation.md
│   └── greeting.md
├── architecture/README.md
├── game-loop/README.md
├── api-spec/README.md
├── frontend/README.md
├── tech-specs/README.md
│   ├── v0.1-mvp.md
│   └── v0.2-multiplayer.md
├── decisions/README.md
├── status/README.md
└── source-map/README.md
```

Каждый README.md содержит:
- Заголовок темы
- «TBD: Будет наполнено из чатов DeepSeek»
- Ссылки на связанные разделы

### Шаг 3: Обновить project_memory
- Запись в `progress.log`
- Обновить `ORIGIN.md` (упомянуть ADND проект)
