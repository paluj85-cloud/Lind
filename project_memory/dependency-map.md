# Карта зависимостей

В этом файле описано, какой файл/раздел на что влияет и от чего зависит.
Использовать перед КАЖДЫМ изменением файлов — чтобы понимать, что ещё нужно будет поправить.

---

## Архитектор → Инженер

| Файл логики (adnd-knowledge/) | Влияет на код | Тип зависимости |
|-------------------------------|---------------|-----------------|
| system-prompts/exis-master.md | backend/app/prompts/exis_master.py | Прямая: промт → шаблон в коде |
| game-design/README.md | backend/app/game/ | Прямая: правила → игровой движок |
| game-loop/README.md | backend/app/game_loop.py | Прямая: цикл игры → реализация |
| api-spec/README.md | backend/app/api/ | Прямая: спецификация → эндпоинты |
| decisions/README.md | Оба трека | Косвенная: архитектурные решения влияют на всё |
| vision/README.md | Все разделы | Косвенная: концепция проекта |
| frontend/README.md | frontend/ | Прямая: дизайн фронтенда → реализация |

## Инженер → Критик

| Код | Что проверять | Инструмент |
|-----|--------------|------------|
| backend/app/api/chat.py | Соответствие game-loop | pytest tests/test_game_loop.py |
| backend/app/prompts/ | Соответствие system-prompts | validate_prompts.py |
| Любое изменение в backend/ | Регресс | pytest tests/ |

## Перекрёстные зависимости (Архитектор ↔ Архитектор)

| Изменение в | Проверить в | Почему |
|------------|-------------|--------|
| system-prompts/ | game-design/ | Не противоречит ли промт дизайну? |
| game-design/ | system-prompts/ | Не сломан ли промт новыми правилами? |
| game-loop/ | system-prompts/ | Поддерживает ли промт новый цикл игры? |
| api-spec/ | frontend/ | Совместим ли фронтенд с новым API? |
| decisions/ | Все разделы | Архитектурные решения → проверка на противоречия |

## Легенда

- **Прямая зависимость**: изменение в источнике ТОЧНО требует изменения в цели
- **Косвенная зависимость**: изменение в источнике МОЖЕТ потребовать изменений в цели — нужно проверить
- **Перекрёстная зависимость**: изменение в одном разделе требует проверки в другом (оба — в Архитекторе)
