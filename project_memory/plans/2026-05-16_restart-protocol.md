# План: Cline Restart Protocol

**Дата**: 2026-05-16 09:31 UTC
**Трек**: Архитектор (конфигурация)
**Задача**: Добавить в `.clinerules` протокол перезагрузки Cline

---

## Проблема

Когда меняется `.clinerules` (или другие конфиги, которые Cline читает при старте), изменения **не вступают в силу** до перезагрузки Cline. При этом после перезагрузки Cline должен знать, на чём мы остановились, и продолжить работу.

Нужен формальный протокол: «Если изменил правила → сохрани состояние → перезагрузи → продолжи».

---

## Варианты решения

### Вариант A: Минимальный (одна строка в Constraints)

Добавить в секцию `## Constraints` один пункт:
```
- ALWAYS update STATE.md and git push BEFORE restarting Cline — so it can resume after reboot.
```

**Плюсы**: Минимум изменений, не раздувает .clinerules
**Минусы**: Нет процедуры — Cline может «забыть» шаги. Нет чёткого условия «когда перезагружать».

---

### Вариант B: Полноценная секция (рекомендую)

Добавить новую секцию `## 🔄 Cline Restart Protocol` с:

1. **When to restart** — чёткие условия (изменения в .clinerules, project_memory/ структуре, system configs)
2. **Procedure (5 шагов)**:
   - Save: git add + commit + push
   - Checkpoint: обновить STATE.md
   - Log: запись в progress.log
   - Announce: сообщить пользователю «Готов к перезагрузке»
   - Restart: пользователь перезагружает → Cline читает STATE.md → продолжает

**Плюсы**: Чёткая процедура, Cline не пропустит шаги. Высокая видимость (отдельная секция).
**Минусы**: +15 строк в .clinerules.

---

## Рекомендация

**Вариант B** — потому что это не одно правило, а процедура из 5 шагов. Одна строка в Constraints не обеспечит надёжность.

---

## Шаги выполнения

1. Добавить секцию `## 🔄 Cline Restart Protocol` в `.clinerules` (перед `## Git Push Safety Rules`)
2. Обновить `project_memory/STATE.md` — актуальный чекпоинт
3. Обновить `project_memory/progress.log`
4. Git commit + push
5. Сообщить пользователю: «Готов к перезагрузке Cline»
6. После перезагрузки: Cline читает `.clinerules` (новое правило вступает в силу) → читает `STATE.md` → продолжает

---

## Что НЕ меняется

- Startup Check уже включает чтение `ORIGIN.md` и `STATE.md`
- Pipeline (Phase 1-5) без изменений
- Все остальные секции `.clinerules` без изменений
