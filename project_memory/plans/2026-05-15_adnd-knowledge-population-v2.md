## Архитектура процесса

```
Этап 0 (Тест): 3 чата → калибровка промтов
    ↓ (промты откалиброваны, ≥95% точность)
Этап 1: Классификация 115 чатов → _classification.json
    ↓
Этап 2: Извлечение фактов партиями по 10 чатов
    ├── Партия 1 (чаты 1-10)  → извлечь → самоаудит ≥95% ✓ → едем дальше
    ├── Партия 2 (чаты 11-20) → извлечь → самоаудит ≥95% ✓ → едем дальше
    ├── ...
    └── Партия 12 (чаты 111-115) → извлечь → самоаудит ≥95% ✓
    ↓
Этап 3: Глобальная консолидация (все 115 чатов, хронологически)
    ↓
Этап 4: Финальная валидация (10% случайных фактов)
    ↓
Этап 5: Документирование
```

## Механика чекпоинтов (каждые 10 чатов)

1. После извлечения фактов из 10 чатов — пауза
2. Выбрать 15% случайных фактов из этой партии (min 5)
3. Для каждого: найти source_quote в исходном чате, сравнить
4. **Если точность ≥95%** → партия принята, едем дальше
5. **Если точность <95%** → анализируем ошибки, правим промт извлечения, переделываем эти 10 чатов с новым промтом

## Этап 0: Тестовый прогон (3 чата)

### Процесс
1. Прочитать 3 чата
2. Классифицировать
3. Извлечь факты (для 1351-строчного — без chunking, влазит в контекст)
4. Сконсолидировать в тестовые `_audit.json`
5. Проверить точность: сравнить 100% фактов с source_quote
6. Скорректировать промты Этапов 1-2

### Критерий готовности
- ≥95% фактов подтверждены source_quote
- Промты скорректированы
- Разрешён переход к Этапу 1

---

## Этап 1: Классификация 115 чатов

### Процесс
1. Cline читает `chat_deepseek/_index.md`
2. Для каждого чата: читает заголовок + поиск ключевых слов по ВСЕМУ файлу
3. Вызывает DeepSeek API (flash, JSON mode) с промтом классификации
4. Результат: `chat_deepseek/_classification.json`

### Формат результата
```json
{
  "classified_at": "2026-05-15",
  "chats": {
    "c723960f": {
      "sections": {
        "vision": "medium",
        "game-design": "none",
        "game-loop": "none",
        "api-spec": "none",
        "frontend": "none",
        "architecture": "none",
        "decisions": "high",
        "status": "low"
      }
    }
  }
}
```

### Критерий готовности
- Каждый из 115 чатов привязан минимум к одному разделу
- Спорные (confidence=low для всех разделов) помечены для ручной проверки

## Этап 2: Извлечение фактов (партии по 10 чатов + чекпоинты)

### Chunking-правила (уточнённые)
- Чат ≤ 2000 строк: один запрос (влазит в контекст DeepSeek flash с резервом на ответ)
- Чат > 2000 строк: разбивка на чанки по 2000 строк с перекрытием 300 строк
- Каждый чанк обрабатывается независимо
- Факты от всех чанков объединяются, дубликаты устраняются на Этапе 3

### Процесс на партию (10 чатов)
1. Для каждого чата в партии:
   - Если ≤ 2000 строк → подать полный текст + промт извлечения
   - Если > 2000 строк → разбить на чанки, подать каждый чанк
2. Cline собирает JSON-ответы в `_audit.json` каждого раздела
3. Параллельно обновляется `_source_map.json`

### Чекпоинт (после каждых 10 чатов)
1. Выбрать 15% фактов (min 5) случайным образом
2. Для каждого: grep source_quote в исходном чате
3. Оценить точность (exact match + semantic match)
4. **≥95%** → продолжаем. **<95%** → анализ ошибок → правка промта → переделка партии.

### Критерий готовности
- Для каждого раздела собран `_audit.json` + `_source_map.json` со всеми фактами
- Все чекпоинты пройдены с ≥95%

---

## Этап 3: Глобальная консолидация (per-section)

### Процесс на раздел
1. Взять `_audit.json` раздела
2. Отсортировать факты по дате чата (source_date) — хронологически
3. Сгруппировать по подтемам (topic)
4. Устранить дубликаты
5. Разрешить противоречия: поздний факт > ранний
6. Подать LLM с промтом консолидации
7. Сохранить `README.md`

### Структура файлов раздела после Этапа 3
```
adnd-knowledge/{раздел}/
├── README.md          ← Чистые знания для AI-мастера
├── _audit.json        ← Полный аудит-трейл
└── _source_map.json   ← Карта: факт → [чат, строка]
```

### Критерий готовности
- README.md каждого раздела заполнен связным текстом
- README.md чистый — без source trace
- Противоречия разрешены хронологически

---

## Этап 4: Финальная валидация

### Процесс
1. Выбрать 10% случайных фактов из каждого раздела (min 3)
2. Для каждого: найти исходный чат, проверить цитату
3. При расхождении: пометить, исправить
4. При системных ошибках: вернуться на Этап 2 с уточнённым промтом

### Критерий готовности
- ≥95% проверенных фактов подтверждены исходными чатами

---

## Этап 5: Документирование

1. Обновить `project_memory/CHANGELOG.md`
