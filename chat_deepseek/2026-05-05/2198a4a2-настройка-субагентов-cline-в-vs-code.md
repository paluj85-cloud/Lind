Привет! Cline в VS Code действительно поддерживает параллельный запуск нескольких субагентов (subagents) для исследования кодовой базы без засорения контекста основного агента.

### 🧠 Зачем нужны субагенты

Представьте, что вам нужно быстро изучить большой проект. Вместо того чтобы тратить время и забивать контекст основного агента, вы можете запустить нескольких субагентов. Каждый из них получит своё задание, например, "исследуй систему аутентификации", "найди все модели баз данных", "проверь файлы конфигурации". Они сделают это параллельно, и вы получите несколько отчётов одновременно.

### 🔧 Как настроить и включить субагентов

Субагенты — это экспериментальная функция, поэтому по умолчанию она выключена. Включить её очень просто:

*   **Включение в настройках**: Откройте настройки Cline в VS Code, кликнув на иконку с шестерёнкой в панели расширения. Перейдите во вкладку **"Features"** (или "Agent"). Найдите переключатель **"Subagents"** и активируйте его[reference:0][reference:1][reference:2]. Эта настройка является общей для всех редакторов (VS Code, JetBrains, CLI)[reference:3].
*   **Запуск командой в чате**: Cline не запускает субагентов по своему усмотрению. Вы должны явно попросить его об этом в своём промпте, например[reference:4]:
    > «Используй субагентов, чтобы исследовать, как реализована аутентификация в проекте и где лежат модели баз данных»

    Другой пример запроса из документации:
    > «Я здесь новичок. Запусти субагентов, чтобы найти главные входные точки приложения, слой маршрутизации и паттерны работы с данными»[reference:5]

### ⚙️ Особенности, ограничения и лучшие практики

*   **Важное ограничение: только для чтения**. Помните, что субагенты не могут вносить изменения в код. Их задача — только исследования. Они могут читать файлы, искать по коду и выполнять команды, не модифицирующие проект (например, `git log`, `ls`), но им **запрещено**[reference:6][reference:7]:
    *   Редактировать и перезаписывать файлы.
    *   Использовать браузер или подключаться к MCP-серверам.
    *   Искать информацию в интернете.
    *   Запускать другие, вложенные субагенты.
*   **Автоодобрение**: Если у вас включено автоматическое одобрение для чтения файлов проекта, Cline не будет спрашивать каждый раз разрешения на запуск субагентов[reference:8].

### 💎 Резюме

Субагенты в Cline — это мощный инструмент для быстрого погружения в незнакомый код или проведения комплексного анализа. Это экспериментальная функция, которая включается одной кнопкой в настройках и запускается по прямому запросу. Просто помните, что они доступны только для чтения, и используйте их для исследовательских задач.

### ⚡️ Способ 1: Быстрая команда «Developer: Reload Window»

Это самый простой и действенный метод, который советуют сами разработчики Cline[reference:0]. По сути, это «мягкая» перезагрузка окна VS Code, которая занимает пару секунд. Все ваши открытые файлы и папки останутся на месте.

1.  Нажмите `F1` или комбинацию `Ctrl+Shift+P` (`Cmd+Shift+P` на macOS).
2.  В появившейся командной строке введите: `Developer: Reload Window`.
3.  Нажмите `Enter`.

### ⏳ Способ 2: Настройка существующих тайм-аутов

У Cline есть встроенные ограничения по времени на выполнение некоторых операций. Если вы сталкиваетесь с зависаниями, можно попробовать увеличить эти значения.

*   **Лимит в 30 секунд на команды**: Cline принудительно завершает любые команды, которые выполняются дольше **30 секунд** (например, сборка проекта или линтинг)[reference:1]. На данный момент это жесткое ограничение, которое нельзя изменить в настройках. Если ваш проект требует долгих операций, об этом стоит знать заранее и разбивать их на более мелкие шаги.

*   **Тайм-аут MCP-серверов**: Если вы используете MCP-серверы (Model Context Protocol), для них можно настроить отдельный тайм-аут ожидания ответа[reference:2].
    1.  Откройте настройки Cline, нажав на иконку шестеренки в панели расширения.
    2.  Перейдите во вкладку **MCP Server Management**.
    3.  Найдите нестабильный сервер и измените значение в поле **Network Timeout**. По умолчанию оно составляет 1 минуту, но его можно увеличить[reference:3].

### 🤖 Способ 3: Автоматизация с расширением `cline-auto-workflow`

Если вы хотите полностью автоматизировать процесс и запускать задачи в Cline при старте VS Code, можно воспользоваться сторонним расширением **`cline-auto-workflow`** из магазина расширений[reference:4]. Этот способ подходит для продвинутых сценариев, но требует осторожности.

1.  Установите расширение `cline-auto-workflow`.
2.  Оно использует таймер (по умолчанию с задержкой в 5 секунд) для координации запуска Cline[reference:5]. Это может помочь, если Cline не успевает загрузиться вместе с редактором.
