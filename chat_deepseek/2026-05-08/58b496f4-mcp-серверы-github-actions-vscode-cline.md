### 🛠️ Детальный обзор каждого MCP-сервера

#### 1. 🌐 GitHub MCP Server (официальный)
Это самый универсальный и простой в настройке сервер от GitHub. Он позволяет ассистенту управлять практически всеми аспектами вашего проекта на GitHub: от просмотра кода до анализа уязвимостей и управления CI/CD[reference:19][reference:20].

**Как настроить в Cline (VS Code):**
*   Поддерживает удаленное подключение, что упрощает настройку.
*   В файле `mcp.json` вашего проекта (или в настройках Cline) добавьте блок `mcpServers`:

```json
{
  "mcpServers": {
    "github": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ЗАМЕНИТЕ_НА_ВАШ_ТОКЕН"
      }
    }
  }
}
```

*   Если вы используете VS Code и предпочитаете OAuth, настройка еще проще. В файле `.vscode/mcp.json` пропишите:

```json
{
  "servers": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/"
    }
  }
}
```

#### 2. 💪 github-repos-manager-mcp
Если вам нужен максимальный охват (целых 89 инструментов!), этот локальный сервер — отличный выбор. Он покрывает управление репозиториями, Issues, Pull Requests, коммитами и ветками[reference:21]. Не требует Docker, что делает его легковесным[reference:22].

**Как настроить:**
*   Установите пакет: `npm install -g github-repos-manager-mcp`
*   Добавьте в `mcp.json`:
```json
{
  "mcpServers": {
    "github-repos-manager": {
      "command": "npx",
      "args": ["-y", "github-repos-manager-mcp"],
      "env": {
        "GITHUB_TOKEN": "ЗАМЕНИТЕ_НА_ВАШ_ТОКЕН"
      }
    }
  }
}
```

#### 3. 🎯 github-actions-mcp-server
Этот сервер создан специально для глубокого управления GitHub Actions. С его помощью можно прямо из VS Code создавать новые workflow, запускать их, просматривать логи и управлять процессом выполнения[reference:23].

**Пример использования:**
Вы можете попросить Cline: *"Создай workflow для запуска тестов при каждом пуше в ветку main и сразу запусти его"*. Cline, используя этот MCP, сгенерирует YAML-файл, создаст коммит и инициирует запуск.

**Как настроить:**
*   Склонируйте репозиторий: `git clone https://github.com/onemarc/github-actions-mcp-server.git`
*   Перейдите в папку и установите зависимости: `npm install && npm run build`
*   Добавьте в `mcp.json`:
```json
{
  "mcpServers": {
    "github-actions-mcp-server": {
      "command": "node",
      "args": ["/ПОЛНЫЙ_ПУТЬ/К/ПАПКЕ/build/index.js"],
      "env": {
        "GITHUB_TOKEN": "ЗАМЕНИТЕ_НА_ВАШ_ТОКЕН"
      }
    }
  }
}
```

#### 4. 🩺 github-actions-mcp
Этот сервер — легковесный «доктор» для ваших пайплайнов. Его единственная задача — быстро извлечь и показать логи из упавшего workflow-запуска, чтобы не нужно было лазить по веб-интерфейсу[reference:24].

**Как настроить:**
*   Не требует установки, запускается через `npx`.
*   Добавьте в `mcp.json`:
```json
{
  "mcpServers": {
    "github-actions-mcp": {
      "command": "npx",
      "args": ["-y", "github-actions-mcp"],
      "env": {
        "GITHUB_TOKEN": "ЗАМЕНИТЕ_НА_ВАШ_ТОКЕН"
      }
    }
  }
}
```

#### 5. 🏢 GitHub Enterprise MCP Server
Незаменимый инструмент, если ваша компания использует GitHub Enterprise Server. Он позволяет взаимодействовать с API вашего приватного инстанса GHES[reference:25].

Настройка требует клонирования и сборки проекта, а также указания URL вашего корпоративного сервера[reference:26]. Детальные шаги лучше смотреть в README соответствующего репозитория.

#### 6. 🐞 Cline Community MCP
Этот сервер помогает правильно оформить баг-репорт или предложение по улучшению для проекта Cline. Он автоматически собирает информацию о системе и версии расширения[reference:27].

**Как настроить:**
*   Склонируйте репозиторий: `git clone git@github.com:cline/cline-community.git`
*   Установите зависимости и соберите: `npm install && npm run build`
*   Добавьте в `mcp.json`:
```json
{
  "mcpServers": {
    "cline-community": {
      "command": "node",
      "args": ["/ПОЛНЫЙ_ПУТЬ/К/ПАПКЕ/build/index.js"]
    }
  }
}
```

#### 7. 🪝 hooks-mcp
Этот сервер позволяет запускать локальные скрипты и команды (тесты, линтеры, билды) через интерфейс MCP, по аналогии с тем, как запускаются шаги в GitHub Actions[reference:28].

**Как настроить:**
*   Установите: `uv tool install hooks-mcp`
*   Опишите нужные команды в YAML-файле[reference:29]
*   Добавьте в `mcp.json`:
```json
{
  "mcpServers": {
    "hooks-mcp": {
      "command": "uvx",
      "args": ["hooks-mcp"]
    }
  }
}
```

Для работы всех этих серверов используется ваш GitHub Personal Access Token. **Крайне важно** создавать токены с минимально необходимыми правами, ограничивать их срок действия и областью применения (конкретными репозиториями)[reference:30]. Никогда не используйте токены с полным доступом ко всем вашим репозиториям.

### 1. Что такое OAuth? 🔐

Если кратко, **OAuth — это стандарт авторизации, который позволяет предоставить стороннему приложению ограниченный доступ к вашим данным на каком-то сервисе, не передавая ему ваш логин и пароль**. OAuth расшифровывается как **Open Authorization**[reference:0].

Представьте, что вы хотите дать приложению доступ к вашим репозиториям на GitHub. Вместо того чтобы сообщать приложению свой пароль, вы используете OAuth:

1.  **Запрос доступа:** Приложение перенаправляет вас на сайт GitHub.
2.  **Подтверждение:** Вы входите в свой аккаунт на самом GitHub и видите список прав, которые запрашивает приложение (например, «только чтение репозиториев»).
3.  **Разрешение:** Вы даёте согласие.
4.  **Доступ:** GitHub выдаёт приложению специальный «ключ» (токен), с помощью которого оно может выполнять только те действия, которые вы разрешили[reference:1].

Такой подход значительно безопаснее, так как ваш пароль остаётся только у вас, а скомпрометированный токен можно быстро отозвать, не меняя пароль[reference:2].

### 2. Связь `api.githubcopilot.com/mcp/` и GitHub Copilot 🤝

Это отличный вопрос, который затрагивает суть интеграции. Давайте разберем два момента:

*   **Что это за адрес?**
    `https://api.githubcopilot.com/mcp/` — это URL **публичного, удаленного GitHub MCP Server**, который размещен и поддерживается самой компанией GitHub[reference:3][reference:4]. Его главное преимущество — **простота настройки**. Вместо того чтобы скачивать, собирать и запускать сервер локально, вы просто указываете этот URL в настройках вашего MCP-клиента (например, Cline в VS Code)[reference:5].

*   **Почему в адресе `copilot`, если это не агент?**
    Ваше замешательство абсолютно логично. GitHub Copilot — это не только AI-агент, но и целая **инфраструктура**, которую GitHub развивает. Использование домена `api.githubcopilot.com` для MCP-сервера — это часть стратегии интеграции.
    
    Хотя MCP-сервер и Copilot — это разные компоненты, они тесно связаны общей экосистемой. Этот сервер обеспечивает **нативную OAuth-аутентификацию**, которая изначально создавалась в том числе для IDE с поддержкой Copilot (VS Code, Visual Studio, JetBrains и др.)[reference:6][reference:7]. Такой подход позволяет:
    *   **Упростить подключение:** Использовать OAuth-авторизацию без ручного создания и копирования токенов доступа (Personal Access Token)[reference:8].
    *   **Стандартизировать работу:** Предоставить единый и удобный способ взаимодействия с GitHub для разных AI-инструментов, включая внешних агентов, таких как Cline.

Таким образом, адрес указывает на инфраструктуру GitHub Copilot, которая используется для предоставления удобного и безопасного сервиса (MCP-сервера) для разных AI-клиентов. Вы можете использовать этот сервер с любым совместимым инструментом, включая Cline, и это не делает его частью AI-агента Copilot.

Если появятся вопросы по настройке такого подключения, спрашивайте, я помогу.

### 🔧 Как это использовать с Cline?

Хотя эти руководства ориентированы на Copilot, base URL `https://api.githubcopilot.com/mcp/` — это просто точка входа для MCP-совместимого клиента, которым и является Cline. Поэтому инструкции по установке для VS Code к вам полностью применимы.

**Пошаговая настройка подключения в VS Code с Cline:**

1.  **Откройте палитру команд:** `Ctrl+Shift+P` (Win/Linux) или `Cmd+Shift+P` (Mac).
2.  **Добавьте сервер:** Начните вводить и выберите команду `MCP: Add Server...`[reference:6].
3.  **Выберите тип:** Выберите `HTTP (HTTP or Server-Sent Events)`.
4.  **Укажите URL:** Вставьте `https://api.githubcopilot.com/mcp/` в поле "Server URL" и нажмите `Enter`[reference:7].
5.  **Авторизация:** Появится запрос на авторизацию через OAuth. Нажмите `Allow` и войдите в свой GitHub аккаунт, если потребуется[reference:8].
6.  **Готово:** Сервер подключен и готов к использованию.
