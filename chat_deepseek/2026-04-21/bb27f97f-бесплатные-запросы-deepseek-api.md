### ⚙️ Технические лимиты

При использовании API важно учитывать и технические ограничения (rate limits):

*   **Частота запросов (RPM)**: Для бесплатных аккаунтов лимит может варьироваться. В некоторых источниках указывается **20 запросов в минуту**[reference:14][reference:15], в других — **50 запросов в минуту**[reference:16][reference:17].
*   **Дневной лимит (RPD)**: Общее количество запросов в день также может быть ограничено, например, **1000 запросов в день**[reference:18][reference:19].
*   **Квоты на токены**: Помимо количества запросов, может быть ограничен и объём обрабатываемых токенов в час. Одна из цитируемых статей упоминает лимит в **50 000 входных и 25 000 выходных токенов в час** для бесплатного уровня[reference:20].

Она предоставляет доступ к актуальным моделям V3 и R1, а также к таким функциям, как веб-поиск и загрузка документов, и не имеет жёстких лимитов на количество сообщений[reference:23].

### RESPONSE (ИИ)

Да, тратятся и те, и другие. Каждый запрос к API DeepSeek суммирует затраты как на отправленный вами **входной** (input) промпт, так и на сгенерированный моделью **выходной** (output) ответ. Официальная документация DeepSeek прямо указывает: *"Мы будем выставлять счёт на основе общего количества входных и выходных токенов, обработанных моделью"*[reference:0].

Другими словами, формула стоимости одного запроса выглядит так:
**(Цена за миллион входных токенов * количество входных токенов) + (Цена за миллион выходных токенов * количество выходных токенов)**[reference:1][reference:2].

### 💎 Почему это важно?

Хотя платят за оба типа токенов, **выходные токены (ответ ИИ) обычно стоят дороже входных**. Это связано с тем, что генерация нового текста требует больше вычислительных ресурсов, чем просто обработка входящего запроса[reference:3][reference:4].

Например, согласно документации DeepSeek, модель `deepseek-chat` стоит:
*   **$0.28 за 1 млн входных токенов** (при промахе кэша)[reference:5]
*   **$0.42 за 1 млн выходных токенов**[reference:6]

Как видите, стоимость выходных токенов примерно в **1.5 раза выше**.

Поэтому, когда вы планируете свой бюджет или анализируете расход бесплатных токенов, стоит учитывать, что итоговая сумма будет складываться из стоимости вашего запроса и сгенерированного ответа, причём генерация ответа вносит в неё основной вклад.

Посчитать токены в DeepSeek API можно несколькими способами: от быстрых приблизительных оценок до точного подсчёта через API.

### 🧮 Способ 1: Приблизительная оценка по символам (самый простой)

Этот способ хорош для быстрой, черновой оценки, но помните — он **не является точным**. Окончательный счёт всегда формирует модель на сервере[reference:0].

В официальной документации DeepSeek приведены следующие соотношения для приблизительной оценки[reference:1][reference:2]:
*   1 английский символ ≈ **0.3 токена** (или около 1 токена на 3-4 символа).
*   1 китайский иероглиф ≈ **0.6 токена** (или около 1 токена на 1.5-2 иероглифа).

Например, фраза "Hello world!" из 12 символов и пробела будет равна примерно 3-4 токенам (12 * 0.3). А фраза "Привет мир" из 9 иероглифов — примерно 5-6 токенам (9 * 0.6).

### ⚙️ Способ 2: Точный подсчёт перед отправкой запроса (Offline)

Если вам нужно узнать точное количество токенов ещё до отправки запроса, DeepSeek предоставляет официальное решение — **офлайн-токенизатор**[reference:3][reference:4].

Это код, который работает локально на вашем компьютере. Он скачивается в виде ZIP-архива со страницы официальной документации DeepSeek[reference:5]. После запуска он преобразует ваш текст в токены и покажет их точное количество[reference:6][reference:7]. Это наиболее надёжный способ для точного планирования бюджета.

### 📈 Способ 3: Просмотр точного расхода после отправки запроса (API)

Самый надёжный способ — посмотреть фактический расход токенов в ответе от API после каждого запроса[reference:8][reference:9]. Именно на основе этих данных происходит списание средств.

В ответе API вы найдёте объект `usage`, который содержит точную информацию[reference:10]:

*   **`prompt_tokens`**: Количество токенов в вашем запросе (входные данные)[reference:11].
*   **`completion_tokens`**: Количество токенов в ответе модели (выходные данные)[reference:12].
*   **`total_tokens`**: Общее количество токенов (`prompt_tokens` + `completion_tokens`)[reference:13].

Кроме того, платёжной единицей, по которой происходит списание, является поле в HTTP-заголовке ответа **`x-billed-tokens`**. В нём содержится итоговое количество токенов, которое будет списано с вашего баланса[reference:14].

### 💡 Другие полезные инструменты

*   **Веб-токенизаторы**: Существуют онлайн-инструменты, например, **Tiktokenizer**, которые могут наглядно показать, как текст разбивается на токены для различных моделей, включая DeepSeek[reference:15][reference:16].
*   **Калькуляторы стоимости**: Некоторые сайты предлагают сервисы, где можно вставить текст и получить примерную оценку стоимости запроса к API[reference:17][reference:18].
*   **Консоль DeepSeek**: На странице вашего аккаунта в разделе "Мониторинг" можно увидеть подробную статистику по всем вашим запросам, включая количество использованных токенов[reference:19].

### 📝 Резюме

1.  **Для грубой оценки** используйте формулы: **1 английский символ ≈ 0.3 токена**, **1 китайский иероглиф ≈ 0.6 токена**.
2.  **Для точного подсчёта до отправки запроса** скачайте и запустите официальный офлайн-токенизатор от DeepSeek.
3.  **Для получения точных цифр после отправки запроса** всегда смотрите поля `usage` и `x-billed-tokens` в ответе от API.

### 🧹 Способ 1: Полный перезапуск (чистая сессия)

Этот подход полностью сбрасывает состояние интерпретатора. Идеально подходит, когда нужно гарантированно избавиться от «мусора» в переменных или проверить код с нуля.

*   **В терминале / командной строке**: самый простой и распространённый способ — завершить текущую сессию и запустить её заново[reference:0]:
    *   **Чтобы выйти**:
        *   **Windows**: Нажмите `Ctrl + Z`, затем `Enter`[reference:1].
        *   **macOS / Linux**: Нажмите `Ctrl + D`[reference:2].
        *   **Универсально**: Введите команду `exit()`[reference:3].
    *   **Чтобы запустить снова**: Просто введите `python` (или `python3`) и нажмите Enter[reference:4].

*   **Программный перезапуск**: если нужно перезапустить программу из неё самой, можно использовать `os.execv()` (этот метод полной замены текущего процесса на новый более сложный и не всегда подходит для интерактивной работы)[reference:5].

*   **В Jupyter Notebook / JupyterLab**: перезапуск «ядра» (kernel) — это стандартная практика, которая очищает все переменные.
    *   Через меню: `Kernel` -> `Restart` или `Restart & Clear Output`[reference:6][reference:7].
    *   **Горячие клавиши**:
        *   **Windows/Linux**: `Ctrl + .`[reference:8]
        *   **macOS**: `Cmd + .`[reference:9]

*   **В средах разработки (IDE)**:
    *   **PyCharm**: Используйте пункт меню `Run` -> `Restart Interpreter`[reference:10].
    *   **VS Code**: Откройте палитру команд (`Ctrl+Shift+P`), найдите и выполните команду `Python: Restart Language Server`[reference:11][reference:12].

### 🔄 Способ 2: Перезагрузка модулей (без остановки)

Этот вариант подходит для разработки, когда вы изменили код модуля и хотите увидеть изменения в текущей сессии, не перезапуская её целиком.

*   **Главный инструмент**: Функция `importlib.reload()`. Она перезагружает указанный модуль из файла, применяя все сделанные в нём изменения[reference:13].
    ```python
    import importlib
    import my_module  # Предположим, ваш модуль называется my_module

    # ... вносите изменения в my_module.py ...

    # Теперь перезагружаем модуль
    importlib.reload(my_module)
    ```
    После этого можно вызывать обновлённые функции из `my_module`.

*   **Полезные советы**:
    *   **IPython**: Для интерактивной работы в IPython есть удобная магия `%autoreload`. Её нужно активировать, и тогда IPython будет автоматически перезагружать модули перед выполнением вашего кода[reference:14].
    *   **Сброс переменных**: В IPython также есть команда `%reset`, которая очищает все переменные в текущей сессии, но не перезагружает модули[reference:15].
    *   **Важно знать**: Функция `reload()` перезагружает **только указанный модуль**. Если ваш модуль импортирует другие, они перезагружены **не будут**[reference:16]. Кроме того, перезагрузка модулей в сложных проектах может привести к ошибкам, например, если у вас есть старые объекты, созданные на основе кода из старой версии модуля[reference:17].

### 💎 Итог

Для большинства задач лучше всего подойдёт **полный перезапуск**. Это самый простой и надёжный способ, который гарантирует, что вы работаете в чистом окружении. `importlib.reload()` — это мощный, но более специфичный инструмент для продвинутой разработки.

### 🕵️‍♂️ Точная диагностика окружения

Самый простой способ начать — всегда проверять путь к интерпретатору, который вы используете прямо сейчас. Это даст вам 100% точный ответ.

```bash
which python
```

В идеале, если ваше виртуальное окружение `python_3.9` активно, эта команда должна показывать путь:
`/home/host1358299/wosgame.ru/venv/python_3.9/bin/python`

Однако, судя по вашей версии Python (3.6.8), команда, скорее всего, укажет на системный Python, например, `/usr/bin/python`. Это значит, что окружение **не активировано**.

### 🚀 Как активировать и проверить ваше окружение

1.  **Активируйте окружение**: Выполните команду, которую вы сами указали:
    ```bash
    source /home/host1358299/wosgame.ru/venv/python_3.9/bin/activate
    ```

2.  **Проверьте версию Python**: После активации снова выполните:
    ```bash
    python --version
    ```
    Теперь вы должны увидеть `Python 3.9.x` (или другую ожидаемую версию).

3.  **Проверьте путь к интерпретатору (для уверенности)**:
    ```bash
    which python
    ```
    Результат должен быть таким, как в примере выше — указывать на ваше виртуальное окружение.

### 📋 Дополнительные команды для проверки

*   **Визуальная проверка в терминале**: После успешной активации в начале строки приглашения командной строки (prompt) должно появиться имя вашего окружения в скобках, например, `(python_3.9) user@host:~$`. Это самый быстрый визуальный индикатор[reference:0].
*   **Команда `pip`**: Также полезно проверить, на какой `pip` вы указываете:
    ```bash
    which pip
    ```
    Этот путь также должен вести в вашу виртуальную среду, например, `/home/host1358299/wosgame.ru/venv/python_3.9/bin/pip`[reference:1][reference:2].
*   **Программная проверка**: Для получения полной информации о вашем окружении изнутри Python можно выполнить:
    ```bash
    python -c "import sys; print(sys.executable)"
    ```
    Эта команда напечатает полный путь к интерпретатору Python, который используется в данный момент[reference:3].
