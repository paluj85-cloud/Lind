*   **🔗 Масштабируемость:** Вы можете отправлять сотни или тысячи запросов параллельно (в рамках тарифов), что невозможно сделать руками в чате.

### ⚖️ Основные критерии выбора

Чтобы принять верное решение, стоит отталкиваться от ответов на эти вопросы:

#### 1. Django: Когда нужна надёжная основа для большого проекта
*   **Нужна готовая админ-панель, система аутентификации и ORM «из коробки»?**
    Если да, то вы сэкономите недели, а то и месяцы разработки, используя встроенные возможности Django[reference:35][reference:36].
*   **Планируете создавать крупный, многофункциональный проект?**
    Django следует принципу DRY и отлично подходит для таких проектов, как соцсети, интернет-магазины, новостные порталы или CRM-системы. На нём работают такие гиганты, как Instagram, Pinterest и Spotify[reference:37][reference:38].
*   **Готовы следовать структуре и правилам, которые предлагает фреймворк?**
    Django «opinionated», что означает, что он диктует определённые правила и структуру проекта. Это помогает команде работать слаженно, но ограничивает гибкость, если вы хотите сделать что-то нестандартно[reference:39].

#### 2. FastAPI: Выбор для современных, высокопроизводительных API
*   **Приоритет — производительность и скорость ответа сервера?**
    FastAPI — бесспорный лидер в Python-экосистеме по скорости работы, превосходя Flask и Django в несколько раз за счёт асинхронности[reference:40][reference:41].
*   **Проект в первую очередь разрабатывается как RESTful API или набор микросервисов?**
    FastAPI идеально заточен под создание API с автоматической валидацией данных (благодаря Pydantic), интерактивной документацией (Swagger UI/ReDoc) и поддержкой современных стандартов, таких как OpenAPI[reference:42][reference:43].
*   **Планируете интегрировать модели машинного обучения?**
    Благодаря высокой производительности и удобству создания API-обёрток, FastAPI — популярный выбор для вывода ML-моделей в продакшен[reference:44].

#### 3. Flask: Свобода и минимализм для небольших и кастомных проектов
*   **Хотите полный контроль над архитектурой и компонентами?**
    Flask не навязывает никаких правил. Вы сами решаете, как организовать проект, какую библиотеку для работы с БД использовать и нужна ли вам валидация форм[reference:45][reference:46].
*   **Проект начинается с малого: прототип, лендинг или несложное API?**
    Flask позволяет запустить первый рабочий прототип в кратчайшие сроки. Однако с ростом проекта вам придётся самостоятельно искать и подключать множество расширений для недостающего функционала[reference:47][reference:48].
*   **Ваш приоритет — обучение веб-разработке на Python?**
    Из-за своей простоты и понятности Flask часто рекомендуют новичкам для понимания базовых принципов работы веб-приложений[reference:49].

#### 4. Dash: Интерактивные дашборды без знания фронтенда
*   **Вам нужно визуализировать данные для аналитики, мониторинга или отчётов?**
    Dash создан специально для этого. Он позволяет строить сложные интерактивные дашборды с графиками, кнопками и фильтрами, используя только код на Python, без необходимости писать JavaScript или HTML[reference:50][reference:51].
*   **Вы работаете в области Data Science, финансов или инженерии?**
    Dash тесно интегрирован с библиотекой Plotly и позволяет легко превратить аналитический код в полноценное веб-приложение для конечных пользователей[reference:52].

#### 5. Bottle и свой фреймворк: Для исключительных случаев
*   **Bottle** — это еще более минималистичный фреймворк, чем Flask, который распространяется в виде одного файла[reference:53][reference:54]. Его стоит рассматривать для очень простых задач, например, создать веб-интерфейс для локального скрипта или развернуть простейший прототип в среде с ограничениями.
*   **«Своё приложение»** (обычно на основе ASGI/WSGI сервера) — оправдано только в исключительных случаях, например, для учебных целей или решения сверхспецифичных задач, когда ни один фреймворк не подходит.

### 💎 Итог

Выбор фреймворка — это стратегическое решение, которое зависит от ваших целей. Если вам нужен надёжный и проверенный инструмент для большого проекта, ваш выбор — **Django**. Если вы создаете современное, высоконагруженное API, то **FastAPI** будет вне конкуренции. Для прототипов, небольших приложений или полной свободы действий идеально подойдёт **Flask**. А если ваша цель — красивая и интерактивная визуализация данных, то ваш инструмент — **Dash**.

Надеюсь, этот разбор поможет вам сориентироваться. Если останутся вопросы по какому-то из фреймворков, смело задавайте!

### ⚖️ FastAPI vs. Flask vs. Django: Что выбрать?

Для вашей задачи я бы исключил **Django**. Несмотря на свою мощь, его монолитная структура и синхронная природа будут скорее мешать в создании гибкого и отзывчивого AI-приложения[reference:6].

Сравним двух основных кандидатов:

| Характеристика | ✅ FastAPI (Рекомендуемый выбор) | 🤔 Flask (Альтернативный вариант) |
| :--- | :--- | :--- |
| **Производительность** | **Очень высокая**. Построен на асинхронном ASGI сервере, идеален для длительных запросов к LLM[reference:7]. | **Средняя**. WSGI по умолчанию синхронный, что может замедлить ответ при запросах к ИИ. |

### `backend/deepseek_client.py`

```python
import os
import httpx
from dotenv import load_dotenv
from typing import List, Dict, Optional

load_dotenv()

DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY")
DEEPSEEK_MODEL = os.getenv("DEEPSEEK_MODEL", "deepseek-chat")
DEEPSEEK_API_URL = "https://api.deepseek.com/v1/chat/completions"

class DeepSeekClient:
    def __init__(self):
        if not DEEPSEEK_API_KEY:
            raise ValueError("DEEPSEEK_API_KEY не найден в переменных окружения")
        self.api_key = DEEPSEEK_API_KEY
        self.model = DEEPSEEK_MODEL
        self.client = httpx.AsyncClient(timeout=60.0)

    async def chat_completion(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 500
    ) -> str:
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        payload = {
            "model": self.model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
            "stream": False
        }

        response = await self.client.post(DEEPSEEK_API_URL, json=payload, headers=headers)
        response.raise_for_status()
        data = response.json()
        return data["choices"][0]["message"]["content"]

    async def close(self):
        await self.client.aclose()
```

```python
from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from models import ChatRequest, ChatResponse
from deepseek_client import DeepSeekClient
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

app = FastAPI(title="AD&D AI Master Chat")

# Настройка CORS для фронтенда
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # В продакшене замените на конкретный домен
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Инициализация клиента DeepSeek (один экземпляр на приложение)
deepseek_client = DeepSeekClient()

@app.on_event("shutdown")
async def shutdown_event():
    await deepseek_client.close()

@app.post("/chat", response_model=ChatResponse)
async def chat_endpoint(request: ChatRequest):
    try:
        # Преобразуем Pydantic модели в словари для клиента
        messages = [msg.dict() for msg in request.messages]
        logger.info(f"Получен запрос с {len(messages)} сообщениями")

        response_text = await deepseek_client.chat_completion(
            messages=messages,
            temperature=request.temperature,
            max_tokens=request.max_tokens
        )
        return ChatResponse(response=response_text)
    except Exception as e:
        logger.error(f"Ошибка при обращении к DeepSeek API: {str(e)}")
        raise HTTPException(status_code=500, detail="Ошибка при генерации ответа")

@app.get("/health")
async def health_check():
    return {"status": "ok"}
```

### `frontend/script.js`

```javascript
document.addEventListener('DOMContentLoaded', () => {
    const chatForm = document.getElementById('chatForm');
    const messageInput = document.getElementById('messageInput');
    const chatMessages = document.getElementById('chatMessages');
    const sendButton = document.getElementById('sendButton');
    const loadingIndicator = document.getElementById('loadingIndicator');

    // API URL (замените на ваш при деплое)
    const API_URL = 'http://localhost:8000/chat';

    // Храним историю сообщений для контекста
    let conversationHistory = [
        { role: 'system', content: 'Ты — опытный Мастер Подземелий для игры AD&D. Твоя задача — вести захватывающее повествование, описывать окружение, отыгрывать NPC и реагировать на действия игрока в духе классического Dungeons & Dragons. Будь креативным, но следуй правилам мира фэнтези.' }
    ];

    // Функция для добавления сообщения в чат
    function addMessageToChat(role, content) {
        const messageDiv = document.createElement('div');
        messageDiv.className = `message ${role}`;

        const avatarDiv = document.createElement('div');
        avatarDiv.className = 'avatar';
        avatarDiv.innerHTML = role === 'user' 
            ? '<i class="fas fa-user"></i>' 
            : '<i class="fas fa-dragon"></i>';

        const contentDiv = document.createElement('div');
        contentDiv.className = 'message-content';
        
        // Преобразуем переносы строк в <p> для красоты
        const paragraphs = content.split('\n').filter(p => p.trim() !== '');
        if (paragraphs.length === 0) {
            contentDiv.innerHTML = '<p>...</p>';
        } else {
            paragraphs.forEach(p => {
                const pElem = document.createElement('p');
                pElem.textContent = p;
                contentDiv.appendChild(pElem);
            });
        }

        messageDiv.appendChild(avatarDiv);
        messageDiv.appendChild(contentDiv);
        chatMessages.appendChild(messageDiv);

        // Автоскролл вниз
        chatMessages.scrollTop = chatMessages.scrollHeight;
    }

    // Функция для отображения ошибки
    function showError(message) {
        const errorDiv = document.createElement('div');
        errorDiv.className = 'message assistant error';
        errorDiv.innerHTML = `
            <div class="avatar"><i class="fas fa-exclamation-triangle"></i></div>
            <div class="message-content"><p style="color: #ff6b6b;">Ошибка: ${message}</p></div>
        `;
        chatMessages.appendChild(errorDiv);
        chatMessages.scrollTop = chatMessages.scrollHeight;
    }

    // Функция отправки запроса к API
    async function sendMessageToAI(userMessage) {
        setLoading(true);
        try {
            // Добавляем сообщение пользователя в историю
            conversationHistory.push({ role: 'user', content: userMessage });

            const response = await fetch(API_URL, {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json',
                },
                body: JSON.stringify({
                    messages: conversationHistory,
                    temperature: 0.8,
                    max_tokens: 600
                })
            });

            if (!response.ok) {
                const errorData = await response.json().catch(() => ({}));
                throw new Error(errorData.detail || `Ошибка сервера: ${response.status}`);
            }

            const data = await response.json();
            const aiResponse = data.response;

            // Добавляем ответ ассистента в историю
            conversationHistory.push({ role: 'assistant', content: aiResponse });

            // Отображаем ответ
            addMessageToChat('assistant', aiResponse);

        } catch (error) {
            console.error('Ошибка:', error);
            showError(error.message || 'Не удалось связаться с AI Мастером');
        } finally {
            setLoading(false);
        }
    }

    // Управление состоянием загрузки
    function setLoading(isLoading) {
        if (isLoading) {
            sendButton.disabled = true;
            messageInput.disabled = true;
            loadingIndicator.classList.remove('hidden');
        } else {
            sendButton.disabled = false;
            messageInput.disabled = false;
            loadingIndicator.classList.add('hidden');
            messageInput.focus();
        }
    }

    // Обработчик отправки формы
    chatForm.addEventListener('submit', async (e) => {
        e.preventDefault();
        const userMessage = messageInput.value.trim();
        if (!userMessage) return;

        // Отображаем сообщение пользователя
        addMessageToChat('user', userMessage);
        
        // Очищаем поле ввода
        messageInput.value = '';

        // Отправляем запрос к AI
        await sendMessageToAI(userMessage);
    });

    // Фокус на поле ввода при загрузке
    messageInput.focus();
});
```

```python
    start_time = time.time()
    
    # Логируем входящий запрос
    logger.info(f"⬇️  --> {request.method} {request.url.path}")

        try:

            return {"type": "http.request", "body": body}
        request._receive = receive

    response = await call_next(request)
```

```python
import os

from dotenv import load_dotenv
from typing import List, Dict, Optional
import logging
import json

load_dotenv()

DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY")
DEEPSEEK_MODEL = os.getenv("DEEPSEEK_MODEL", "deepseek-chat")
DEEPSEEK_API_URL = "https://api.deepseek.com/v1/chat/completions"

class DeepSeekClient:
    # ... __init__ без изменений ...

    async def chat_completion(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 500
    ) -> str:
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        payload = {
            "model": self.model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
            "stream": False
        }

        # Логируем запрос к DeepSeek (скрываем ключ)
        safe_headers = {**headers, "Authorization": "Bearer ***"}
        logger.info(f"🤖 DeepSeek Request: URL={DEEPSEEK_API_URL}")
        logger.debug(f"Headers: {safe_headers}")

            content = msg.get("content", "")
            preview = content[:100] + "..." if len(content) > 100 else content
            preview_messages.append({**msg, "content": preview})
        logger.info(f"📤 Payload: {json.dumps({**payload, 'messages': preview_messages}, indent=2, ensure_ascii=False)}")

        try:
            response = await self.client.post(DEEPSEEK_API_URL, json=payload, headers=headers)
```

```python
import os

from typing import List, Dict, Optional

import json

load_dotenv()

DEEPSEEK_MODEL = os.getenv("DEEPSEEK_MODEL", "deepseek-chat")
DEEPSEEK_API_URL = "https://api.deepseek.com/v1/chat/completions"

class DeepSeekClient:
    def __init__(self):
        if not DEEPSEEK_API_KEY:
            raise ValueError("DEEPSEEK_API_KEY не найден в переменных окружения")
        self.api_key = DEEPSEEK_API_KEY
        self.model = DEEPSEEK_MODEL
        self.client = httpx.AsyncClient(timeout=60.0)

    async def chat_completion(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 500
    ) -> str:
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        payload = {
            "model": self.model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
            "stream": False
        }

        # Логируем запрос к DeepSeek (скрываем ключ)
        safe_headers = {**headers, "Authorization": "Bearer ***"}
        logger.info(f"🤖 DeepSeek Request: URL={DEEPSEEK_API_URL}")
        logger.debug(f"Headers: {safe_headers}")
        
        # Обрезаем длинные сообщения для лога
        preview_messages = []
        for msg in messages:
            content = msg.get("content", "")
            preview = content[:100] + "..." if len(content) > 100 else content
            preview_messages.append({**msg, "content": preview})
        logger.info(f"📤 Payload: {json.dumps({**payload, 'messages': preview_messages}, indent=2, ensure_ascii=False)}")

        try:
            response = await self.client.post(DEEPSEEK_API_URL, json=payload, headers=headers)
            response.raise_for_status()
            data = response.json()
            
            # Логируем ответ
            usage = data.get("usage", {})
            logger.info(f"📥 DeepSeek Response: status={response.status_code}, "
                       f"tokens: prompt={usage.get('prompt_tokens')}, "
                       f"completion={usage.get('completion_tokens')}, "
                       f"total={usage.get('total_tokens')}")
            
            content = data["choices"][0]["message"]["content"]
            preview_content = content[:200] + "..." if len(content) > 200 else content
            logger.info(f"💬 Response content preview: {preview_content}")
            
            return content
        except httpx.HTTPStatusError as e:
            logger.error(f"❌ DeepSeek HTTP error: {e.response.status_code} - {e.response.text}")
            raise
        except Exception as e:
            logger.error(f"❌ DeepSeek request failed: {str(e)}")
            raise

    async def close(self):
        await self.client.aclose()
```

### 🚧 Почему текущий подход не работает на виртуальном хостинге HostLand

Ваш план столкнулся с несколькими фундаментальными ограничениями, которые типичны для большинства тарифов виртуального хостинга.

1.  **Главное ограничение: Нет доступа к запуску процессов.** На виртуальном хостинге вы не можете запускать свои программы как отдельные, постоянно работающие процессы. Ваш скрипт на Python с Uvicorn как раз и является таким процессом. Хостинг просто не даст ему работать в фоне.

2.  **Ошибка `ERR_CONNECTION_REFUSED`:** Она возникает именно из-за пункта 1. Ваш браузер пытается достучаться до порта `8000`, но на сервере там ничего не работает, потому что Uvicorn не запущен.

3.  **Ограничение на проксирование (`.htaccess`):** Хотя настройка прокси в `.htaccess` технически возможна, на многих shared-хостингах модуль `mod_proxy` отключен в целях безопасности и стабильности. Поэтому есть большая вероятность, что даже если бы Uvicorn работал, запросы до него бы не дошли.

**Вывод:** К сожалению, развернуть ваше приложение в его текущей архитектуре на виртуальном хостинге **технически невозможно**. Стандартные рецепты, которые работают на VPS или выделенном сервере, здесь неприменимы.

```nginx
        proxy_set_header X-Real-IP $remote_addr;
```

### ⚙️ Правильная настройка FastAPI на HostLand

Для запуска вашего приложения на HostLand не нужно вручную запускать Uvicorn через консоль. Вместо этого используется специальный обработчик в панели управления. Вот пошаговая инструкция:

#### 🧭 Пошаговая инструкция

1.  **Создайте приложение в панели управления:**
    *   Войдите в панель управления HostLand.
    *   Перейдите в раздел управления вашим доменом.
    *   Найдите вкладку **«Python»** (или «Сайты» → «Добавить» → вкладка «Python») и выберите фреймворк **«FastAPI»**[reference:0].

2.  **Настройте параметры приложения:**
    *   **Версия Python:** Выберите нужную версию из выпадающего списка. Рекомендую выбрать Python 3.9 или выше[reference:1].
    *   **Каталог приложения:** Укажите имя для папки, где будет располагаться ваш проект (например, `my-apps`). **Важно:** Это должен быть тот же каталог, куда вы ранее загрузили файлы проекта (`backend` и `frontend`)[reference:2].
    *   **URL приложения:** Это будет ссылка на ваш сайт. Если вы хотите, чтобы приложение было доступно по корневому адресу (например, `http://ваш-домен/`), оставьте это поле **пустым**[reference:3][reference:4].

3.  **Сохраните и подождите:** Нажмите кнопку «Сохранить». HostLand автоматически установит и настроит все необходимое в течение 5-7 минут[reference:5].

#### 🔄 Как теперь управлять приложением?

После такой установки вам не нужно будет вручную запускать сервер через `nohup`. Хостинг сам позаботится о его работе.

*   **Перезагрузка после изменений:** Когда вы будете вносить правки в код, для их применения потребуется перезагрузить приложение. Вместо перезапуска процесса в консоли, вам нужно просто изменить (например, открыть и сохранить) специальный файл: `/home/hostXXXXXXX/domain_name/projects/app_catalog/tmp/restart.txt`[reference:6].
*   **Просмотр логов:** Для отладки смотрите логи ошибок веб-сервера (Apache), которые обычно доступны в панели управления хостингом в разделе «Логи»[reference:7].

### 🔌 Финальный штрих: настройка фронтенда

Теперь, когда бэкенд настроен через панель управления, важно правильно указать к нему путь из вашего JavaScript-файла. Убедитесь, что в `frontend/script.js` строка `API_URL` выглядит так:

```javascript
const API_URL = '/chat';
```

Эта относительная ссылка гарантирует, что запросы с вашего фронтенда будут корректно отправляться на бэкенд, работающий на том же домене.
