### RESPONSE (ИИ)

Привет! Всё верно, классический PowerShell (Windows PowerShell 5.1 и более ранние) не поддерживает оператор `&&`, который привычен для пользователей `bash`, `cmd` или `sh`.

### Почему так сложилось?

1. **Разная философия работы с командами**  
   В `cmd` или `bash` команды возвращают просто код возврата (0 = успех, не 0 = ошибка). Оператор `&&` смотрит на этот код и решает, выполнять ли следующую команду.  
   В PowerShell же команды возвращают **объекты** (не только код выхода), а статус выполнения можно проверить через автоматическую переменную `$?` (была ли ошибка) или `$LASTEXITCODE` (код от внешних программ). `&&` в исходной версии не был реализован, чтобы не создавать путаницу с оператором `-and` для логических выражений (например, `(Test-Path file.txt) -and (Get-Content file.txt)`).

2. **Исполнение конвейера**  
   В PowerShell конвейер (`|`) работает совершенно по-другому — он передаёт объекты между командами, а не просто текст. Внедрение условного оператора `&&` требовало переработки грамматики.

### Что делать вместо `&&`?

В **классическом PowerShell (5.1 и ниже)** обычно используют такие способы последовательного выполнения команд с проверкой успеха:

```powershell
# 1. Просто разделяйте точкой с запятой (всегда выполняются, без проверки)
command1; command2

# 2. С проверкой через $?
if (command1) { command2 }

# Например:
if (ping -n 1 google.com) { ipconfig }

# 3. С проверкой кода возврата внешних программ
command1; if ($LASTEXITCODE -eq 0) { command2 }

# 4. Используя -and внутри одного вызова (только для проверок, не для любых команд)
(command1) -and (command2)   # Выполнит command2 только если command1 вернёт true
```

### А в современном PowerShell (версии 7+)?

**Начиная с PowerShell Core 6 и выше (включая PowerShell 7) операторы `&&` и `||` уже реализованы** — именно как условные цепочки команд, работающие аналогично bash/cmd.

Пример в PowerShell 7:
```powershell
# Выполнит command2 только если command1 успешен
command1 && command2

# Выполнит command2 только если command1 завершился с ошибкой
command1 || command2
```

Так что если вы используете старую версию (`$PSVersionTable.PSVersion`), `&&` не будет работать. Но можете перейти на PowerShell 7 — он обратно совместим с большинством ваших скриптов и добавляет много удобств.

### Краткий итог

- **Windows PowerShell 5.1** — `&&` нет, используйте `if` или `;`.
- **PowerShell 7+** — `&&` и `||` есть, работают как привычно.

Если не хотите обновляться, самый простой способ склеить две команды с проверкой:  
`первая команда; if ($?) { вторая команда }`
