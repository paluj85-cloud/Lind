### 🚀 Как развернуть модель DeepSeek локально: пошаговый план

Самый простой и популярный способ для начинающих — использовать **Ollama**. Этот инструмент берет на себя всю техническую сложность загрузки и запуска моделей. Альтернативный путь для более продвинутых пользователей — **vLLM** с Docker, который обеспечивает более высокую производительность.

Вот пошаговая инструкция для старта:

1.  **Установите Ollama**: Скачайте и установите программу с официального сайта [ollama.com](https://ollama.com) для вашей операционной системы (Windows, macOS, Linux)[reference:3].
2.  **Загрузите и запустите модель**: Откройте терминал (командную строку) и выполните команду для загрузки и запуска нужной модели. Например, для 7B-версии DeepSeek-R1:
    ```bash
    ollama run deepseek-r1:7b
    ```
    Эта команда автоматически загрузит модель и запустит с ней диалог[reference:4].
3.  **Начните использовать (Чат)**: После выполнения команды можно сразу начинать общение с моделью прямо в терминале. Это идеально для быстрого тестирования.
4.  **Начните использовать (API)**: Ollama автоматически запускает локальный API-сервер, совместимый с форматом OpenAI. Вы можете слать к нему запросы из ваших приложений по адресу `http://localhost:11434`[reference:5][reference:6].

### 💻 Какая модель вам подойдет?

Выбор модели напрямую зависит от мощности вашего компьютера. Вот сравнение популярных версий:

| Модель | Рекомендуемые требования | Для каких задач подходит |
| :--- | :--- | :--- |
| **DeepSeek-R1 7B/8B** | **VRAM:** до 8 ГБ<br>**RAM:** от 16 ГБ[reference:7] | Для тестирования, обучения, простых задач и слабых ПК. Отличный старт для знакомства. |
| **DeepSeek-R1 14B/32B** | **VRAM:** 8-24 ГБ<br>**RAM:** от 32 ГБ[reference:8] | Более качественные ответы, решение сложных задач, написание кода. Хороший баланс для мощного домашнего ПК. |
| **DeepSeek-V3/V4 (полные)** | **VRAM:** десятки ГБ (H100)<br>**RAM:** сотни ГБ[reference:9][reference:10] | Только для серверов и энтузиастов с очень мощным оборудованием. Требуют кластеры из видеокарт. |

> **Важно:** Для моделей 7B-32B часто применяется квантизация — сжатие модели для уменьшения потребления памяти с минимальной потерей качества. Ollama делает это автоматически. Модели V3/V4 в полной версии крайне требовательны к ресурсам, и их запуск на домашнем ПК практически невозможен[reference:11].

Запуск полной версии DeepSeek-V4 (671B параметров) — задача, требующая очень серьезных ресурсов. Это не тот случай, когда подойдет обычный VDS (Virtual Dedicated Server). Перед тем как выбирать конкретного хостера, давайте посмотрим на технические требования.

### 💻 Системные требования

**DeepSeek-V4** — это Mixture-of-Experts (MoE) модель. Хотя в ней всего 671 млрд параметров, в каждый момент времени активны лишь около 37 млрд из них[reference:0]. Этот факт сильно влияет на требования к памяти в зависимости от того, как вы оптимизируете запуск.

Вот примерные требования к видеопамяти (VRAM) для запуска DeepSeek-V4:

| Точность вычислений | Вес модели (ГБ) | Активная часть (ГБ) | Примечания |
| :--- | :--- | :--- | :--- |
| **FP16 (половинная точность)** | ~1342 ГБ | ~74 ГБ + накладные расходы | Требует кластер из 8 и более H100/A100. Только для серверов. [reference:1] |
| **FP8 (8-битная)** | ~500-700 ГБ | ~37-46 ГБ | Значительно меньше FP16, но всё равно требует нескольких топовых GPU. [reference:2][reference:3] |
| **INT4 (4-битная)** | ~175 ГБ | ~22-26 ГБ | Самая легкая версия, потенциально может быть запущена на одном мощном GPU (например, H100 80GB). [reference:4] |

> **Важное примечание:** Цифры для полной модели (FP16 и FP8) актуальны, если вы загружаете все 671B параметров в память. Если использовать техники параллелизма (например, `expert parallelism`), требования будут соответствовать строке "активная часть" для более дорогих конфигураций[reference:5]. Однако для большинства облачных развертываний проще арендовать необходимое количество видеокарт.

### ⚖️ Сравнение вариантов развертывания

Важно понимать разницу между подходами. Вы можете не арендовать "голое железо" и настраивать все самостоятельно, а воспользоваться готовым API.

| Характеристика | SaaS-решения (Alibaba Cloud, Hyperbolic, Together AI) | Аренда GPU-сервера (IaaS) |
| :--- | :--- | :--- |
| **Сложность** | ⭐ (низкая) | ⭐⭐⭐ (высокая) |
| **Скорость запуска** | от нескольких минут[reference:27] | от 15 минут до нескольких часов[reference:28] |
| **Контроль** | Только через API | Полный root-доступ |
| **Оплата** | За токены | За время аренды сервера |
| **Гибкость** | Низкая (работаете в рамках API) | Высокая (полный контроль) |

*   **Для быстрого тестирования** идеально подойдут SaaS-платформы, такие как **Alibaba Cloud Model Studio** (ранее Bailian) или **Hyperbolic**.
*   **Для production-среды** с высокой нагрузкой лучше выбрать аренду выделенного сервера (IaaS), например, у **immers.cloud** или **CloudCompute**.
