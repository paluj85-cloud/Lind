### RESPONSE (ИИ)

VS Code не имеет встроенной поддержки FTP, но это легко решается с помощью специальных расширений. Самым популярным и надёжным выбором является расширение **SFTP**, так как оно активно обновляется и имеет хорошую документацию[reference:0].

### ⚙️ Подключение: установка и настройка SFTP

Процесс настройки будет выглядеть так:

1.  **Установка расширения**: Нажмите `Ctrl+Shift+X` (или иконку квадратов слева), в поиске введите `SFTP` и установите расширение от *Natizyskunk*[reference:1][reference:2].
2.  **Подготовка проекта**: Откройте папку вашего локального проекта, которую вы хотите синхронизировать с сервером[reference:3].
3.  **Настройка подключения**:
    *   Откройте командную палитру (`Ctrl+Shift+P` / `Cmd+Shift+P`), введите `SFTP: Config` и выполните команду[reference:4][reference:5].
    *   Эта команда создаст в папке `.vscode` файл конфигурации `sftp.json`[reference:6]. Отредактируйте его, указав свои данные[reference:7]:
        ```json
        {
            "host": "ваш.ftp-сервер.com",
            "port": 21,
            "username": "ваше_имя_пользователя",
            "password": "ваш_пароль",
            "protocol": "ftp",
            "remotePath": "/путь/к/папке/на/сервере",
            "uploadOnSave": true
        }
        ```
        > Параметр `"uploadOnSave": true` автоматически загружает файл на сервер при каждом его сохранении в редакторе[reference:8][reference:9].

### 🚀 Использование и работа с сервером

*   **Открытие удалённой папки**: В командной палитре (`Ctrl+Shift+P`) выберите `SFTP: List Remote Directory`, чтобы просмотреть файлы на сервере.
*   **Синхронизация файлов**: При изменении и сохранении локального файла он автоматически загрузится на сервер (если включена опция `uploadOnSave`). Вы также можете запустить синхронизацию вручную через команду `SFTP: Sync Local -> Remote` в командной палитре[reference:10].
*   **Отслеживание изменений**: Расширение показывает статус синхронизации (например, облачко при автоматической загрузке), поэтому вы всегда будете видеть актуальное состояние[reference:11].

### 💡 Альтернативные расширения

Кроме SFTP, есть и другие расширения. Вы можете ознакомиться с ними в маркетплейсе VS Code, но SFTP является стандартным и самым надёжным решением.

*   **FTP-Simple**: Простое расширение для базовых операций[reference:12].
*   **FTP-Sync**: Расширение для двунаправленной синхронизации[reference:13].
*   **FTP Manager**: Расширение с более широким функционалом для управления FTP[reference:14].

### 🚀 Как подключиться к серверу

1.  **Откройте командную палитру**: Нажмите `Ctrl+Shift+P` (на Windows/Linux) или `Cmd+Shift+P` (на macOS)[reference:0].
2.  **Запустите синхронизацию**: В появившейся строке введите `SFTP: Sync Local -> Remote`. Это команда, которая загружает все ваши локальные файлы на сервер, устанавливая соединение[reference:1].
    *   Альтернативный способ — выполнить команду `SFTP: Download Project` (загрузка проекта с сервера), которая также инициирует подключение.

### 🗂️ Навигация по удаленным файлам

После того как вы подключились, вы можете просматривать файлы на сервере, как если бы они были на вашем компьютере, с помощью встроенного в VS Code файлового менеджера.

*   **Через иконку SFTP**: Слева, на боковой панели (Activity Bar), появится новая иконка **SFTP**. Нажмите на неё, чтобы открыть обозреватель удаленных файлов, где можно будет увидеть всё содержимое вашего сервера[reference:2].
*   **Через командную палитру**: Вы также можете выполнить команду `SFTP: List Remote Directory`, чтобы открыть список файлов на сервере в отдельной вкладке.

После этого вы сможете открывать, редактировать и сохранять файлы прямо на сервере, а все изменения будут синхронизироваться автоматически[reference:3].

Ключевое различие между FTP и SFTP — это **безопасность**. Если провести аналогию, FTP передаёт данные открыто, как на почтовой открытке, а SFTP использует защищённый конверт, шифруя всю информацию[reference:0][reference:1]. Давайте разберём это и другие отличия подробнее.

### 🛡️ Основные различия FTP и SFTP

| Характеристика | FTP (File Transfer Protocol) | SFTP (SSH File Transfer Protocol) |
| :--- | :--- | :--- |
| **Безопасность** | Низкая. Данные, логины и пароли передаются в **открытом виде** и могут быть перехвачены[reference:2]. | Высокая. **Вся** информация (данные, команды, пароли) **шифруется**[reference:3][reference:4]. |
| **Протокол** | Автономный протокол, основанный на TCP/IP[reference:5]. | Часть протокола SSH (Secure Shell)[reference:6]. |
| **Порт(ы)** | Использует несколько портов: 21 для команд и диапазон портов для передачи данных[reference:7][reference:8]. | Использует **один порт** (обычно 22) для всего: и для команд, и для данных[reference:9][reference:10]. |
| **Скорость** | Как правило, быстрее, так как нет накладных расходов на шифрование[reference:11]. | Может быть немного медленнее из-за процесса шифрования и дешифрования данных[reference:12]. |
| **Аутентификация** | Логин/пароль (передаются открыто)[reference:13]. | Логин/пароль или **SSH-ключи** — более безопасный метод. Можно подключиться без ввода пароля[reference:14][reference:15]. |
| **Возобновление передачи** | Поддерживается[reference:16]. | Поддерживается в полной мере, можно приостанавливать и возобновлять передачу файлов[reference:17][reference:18]. |
| **Совместимость** | Поддерживается практически всеми системами и устройствами[reference:19]. | Требуется SSH-сервер, но также широко поддерживается. |

### 📝 Краткий итог

Протокол **FTP** — это быстрый, но устаревший и крайне небезопасный способ передачи файлов. Его можно использовать только внутри полностью доверенных и изолированных сетей[reference:20].

Протокол **SFTP** — это современный, безопасный и рекомендованный стандарт для большинства задач, включая загрузку файлов на веб-хостинг. Он надёжно шифрует все данные и прост в настройке (использует один порт)[reference:21][reference:22].

Подводя итог, для сохранности ваших паролей и проектов всегда стоит выбирать SFTP, когда это возможно. Это особенно важно при работе с удалёнными серверами в интернете.
