### ⚙️ Ключевые моменты реализации

Теперь о том, как воплотить это в жизнь.

1.  **Глобальный поиск и мышление**
    Ты хочешь, чтобы эти функции были включены всегда. В терминах DeepSeek API:
    *   **«Мышление» (Reasoning)**: Это достигается через параметр `thinking`. В запросе к API нужно всегда передавать объект `"thinking": {"type": "enabled"}`.
    *   **«Глобальный поиск»**: Судя по описанию, это Web Search. Он также включается в API.

    **Важно**: На данный момент, согласно документации, DeepSeek **не поддерживает одновременное использование `thinking` и `tools`** (к которым относится поиск в интернете). Придется выбирать что-то одно для каждого запроса.

2.  **Сборка «глобального контекста»**
    Перед каждым запросом к ИИ твой бэкенд должен выполнять SQL-запрос, чтобы найти все сообщения, которые нужно «видеть» модели:
    ```sql
    -- Собираем все глобальные сообщения
    SELECT role, content FROM messages WHERE is_global = true
    UNION ALL
    -- Собираем сообщения из чатов, которые полностью отмечены как глобальные
    SELECT messages.role, messages.content
    FROM messages
    JOIN chats ON messages.chat_id = chats.id
    WHERE chats.is_global = true
    ORDER BY created_at;
    ```
    Найденные сообщения вставляются в самое начало истории диалога, который отправляется в API, вместе с текущим запросом пользователя.

3.  **Стриминг и остановка генерации**
    *   **Стриминг**: При каждом ответе от DeepSeek API данные приходят по частям (chunks). Бэкенд должен пересылать их на фронтенд, используя Server-Sent Events (SSE).
    *   **Остановка генерации**: Когда пользователь нажимает кнопку «Остановить», фронтенд отправляет специальный запрос на бэкенд. Бэкенд в этот момент прерывает стрим-соединение с API DeepSeek. Полученные до этого момента «кусочки» ответа можно сохранить в БД.

4.  **Переписывание сообщения**
    Эта функция не требует удаления старого сообщения из базы данных (чтобы не терять историю). Лучше создать для него «версионирование»:
    *   Добавь в таблицу `messages` поле `edited_content` (текст отредактированного сообщения) и `edited_at` (время редактирования).
    *   Когда пользователь переписывает сообщение, ты сохраняешь его в `edited_content`.
    *   При формировании контекста для ИИ используй `edited_content`, если оно не пустое, иначе `content`.

## Техническое задание на разработку веб-приложения «AI Chat с глобальной памятью»

### 1. Общее описание

Веб-приложение представляет собой чат-интерфейс для взаимодействия с ИИ-моделью DeepSeek. Приложение открывается в браузере смартфона и адаптировано под небольшой экран. Все диалоги, сообщения и настройки сохраняются на сервере в базе данных, поэтому после перезагрузки страницы или перезапуска устройства история не теряется.

Ключевая особенность — **система глобального доступа**: любое сообщение или целый чат можно отметить как «глобальный», после чего эта информация автоматически добавляется в начало каждого нового диалога с ИИ (во всех чатах). Таким образом, ИИ всегда «знает» важные сведения и действует в едином контексте, который формирует сам пользователь.

Встроенные возможности DeepSeek:
- **Мышление (Thinking)** — режим углубленного рассуждения ИИ; включён постоянно для всех запросов.
- **Глобальный поиск (Global Search)** — реализован силами приложения через описанную выше систему глобального доступа (RAG на основе отмеченных сообщений). Внешний интернет-поиск не используется.

### 2. Термины

- **Чат** — отдельная ветка общения, содержащая последовательность сообщений пользователя и ответов ИИ.
- **Глобальное сообщение** — сообщение, отмеченное специальным маркером (галочкой). Такие сообщения автоматически включаются в контекст любого чата перед отправкой нового запроса.
- **Глобальный чат** — чат, у которого установлен флаг «глобальный». Все сообщения такого чата считаются глобальными, если у конкретного сообщения не переопределён этот признак.

### 3. Функциональные требования

#### 3.1. Список чатов (главный экран)

При входе пользователь видит список всех существующих чатов, отсортированный по дате последнего сообщения (сверху — самые свежие).

| Элемент | Поведение |
|--------|-----------|
| Строка чата | Показывает название чата и фрагмент последнего сообщения. При нажатии открывает соответствующий чат. |
| Значок глобальности | Рядом с названием отображается иконка (например, земной шар), если чат отмечен как глобальный. |
| Удаление чата | Выполняется свайпом влево по строке чата с подтверждением, либо через меню внутри чата. |
| Кнопка «+» (создать) | Расположена в правом верхнем углу. Создаёт новый пустой чат с автоматическим именем «Новый чат» и сразу открывает его. |

#### 3.2. Экран чата

Внутри открытого чата расположены:

- **Область сообщений** — вертикальная прокручиваемая лента, занимающая основную часть экрана.
- **Поле ввода** — закреплено внизу экрана. Содержит текстовое поле и кнопку отправки (стрелка или самолётик).
- **Кнопка «Стоп»** — появляется рядом с полем ввода только в момент, когда ИИ генерирует ответ. Нажатие мгновенно останавливает генерацию. Частично полученный текст сохраняется как ответ.
- **Заголовок чата** — в верхней части экрана. Содержит название чата (редактируется по нажатию на него) и кнопку настроек чата (три точки или шестерёнка). В настройках доступны: переключатель «Глобальный доступ ко всему чату» и удаление чата.

**Глобальный доступ в чате:**
- Включение тумблера делает все существующие и будущие сообщения этого чата глобальными. Выключение — отменяет это свойство. Сообщения, у которых глобальный доступ был проставлен индивидуально вручную, не изменяются (остаются глобальными).

**Переписывание сообщения:**
- Долгое нажатие на «своё» сообщение пользователя открывает контекстное меню с опцией «Переписать».
- При выборе опции текст сообщения становится редактируемым, пользователь вносит исправления и подтверждает отправку.
- Старая версия сообщения не теряется, но в истории начинает отображаться новый текст. Связанный с этим сообщением ответ ИИ удаляется (или скрывается).
- После редактирования ИИ получает обновлённый контекст и генерирует новый ответ.

#### 3.3. Отметка отдельных сообщений как глобальных

У каждого сообщения (и вопроса, и ответа ИИ) слева или справа расположена миниатюрная иконка-галочка / земной шар. Нажатие переключает статус глобальности для этого конкретного сообщения.

Состояния иконки:
- Пустая/серая — сообщение не глобально.
- Залитая/цветная — сообщение является глобальным и будет включаться в контекст всех чатов.

Если чат целиком отмечен как глобальный, иконки у всех его сообщений по умолчанию активны, но пользователь может снять галочку с отдельных сообщений, чтобы исключить их из глобального контекста.

### 4. Логика работы глобального контекста (Global Search)

Перед отправкой любого нового запроса в DeepSeek система автоматически выполняет выборку всех сообщений с активным признаком глобальности из всех чатов пользователя.

Эти сообщения (в хронологическом порядке) вставляются в самое начало диалога, формируя псевдо-сообщения от пользователя и ассистента, содержащие глобальную информацию. Затем к ним добавляется текущая история чата, после чего весь пакет направляется в API.

Таким образом, ИИ «видит» глобальный контекст во всех чатах, но не имеет доступа к неотмеченным данным. Обновление контекста происходит мгновенно при любом изменении флагов глобальности.

Режим мышления (Thinking) при этом передаётся в каждом запросе через соответствующий параметр API (без возможности отключения).

### 5. Остановка генерации и поведение при ошибках

- При нажатии «Стоп» во время генерации ответа соединение с API асинхронно разрывается. Накопленный текст ответа немедленно отображается и фиксируется в истории чата.
- В случае обрыва сети или ошибки API на экран выводится уведомление, позволяющее повторить последний запрос. Повторная отправка восстанавливает исходное состояние диалога (без дублирования сообщений).

### 6. Интерфейс и юзабилити (адаптация под смартфон)

Приложение должно выглядеть как нативное мобильное приложение при открытии в браузере телефона.

- Кнопки и интерактивные элементы имеют минимальный размер 44×44 точки для удобства касания пальцем.
- Шрифт основного текста чата достаточно крупный, комфортный для чтения без масштабирования.
- Область ввода зафиксирована, не перекрывает сообщения при появлении клавиатуры.
- Свайповые жесты используются для удаления (с подтверждением).
- Долгое нажатие открывает контекстные действия.
- Цветовая индикация: сообщения пользователя и ИИ визуально разделены (например, разные оттенки фона). Иконки глобальности ненавязчивы, но легко находимы.
- Адаптивная верстка: на очень узких экранах поля и отступы уменьшаются пропорционально, но не теряют функциональности.

### 7. Хранение данных и безопасность

- Все данные (чаты, сообщения, флаги глобальности) хранятся в базе данных на серверной стороне. Никаких данных на устройстве пользователя не остаётся, кроме стандартных кук сессии.
- API-ключ DeepSeek хранится исключительно на сервере, пользователь его не видит.
- Обмен данными между клиентом и сервером идёт по защищённому протоколу HTTPS.
- Аутентификация пользователя (если требуется в будущем) может быть добавлена через любой стандартный метод, но на старте проект может работать как однопользовательское приложение с единой базой.

### 8. Ограничения

- Интерфейс загрузки файлов и картинок отсутствует.
- Одновременная работа режимов «Мышление» и «Интернет-поиск» в API DeepSeek невозможна, поэтому интернет-поиск не задействован. Функция, называемая пользователем «глобальный поиск», реализована исключительно на базе данных приложения (RAG).

### RESPONSE (ИИ)

На сайте `chat.deepseek.com` одновременная работа обоих режимов реализована не через API, где они конфликтуют, а благодаря более сложной внутренней архитектуре самого сайта. Это не просто одновременное включение, а технология под названием **«чередующееся мышление» (Interleaved Thinking)**[reference:0][reference:1].

Суть подхода в том, что это не параллельный, а **циклический процесс**, похожий на работу исследователя[reference:2]. Вы задаёте вопрос, и запускается «исследовательский цикл»:

1.  **Поиск (Search)**: Модель ищет информацию в интернете по вашему запросу.
2.  **Чтение (Read)**: Модель «открывает» и изучает содержимое найденных страниц.
3.  **Мышление (Thinking)**: На основе прочитанного модель делает паузу, чтобы «обдумать» информацию. Например, насколько авторитетен источник, нет ли противоречий и как полученные данные соотносятся с изначальным вопросом[reference:3][reference:4].
4.  **Следующий шаг**: На основе своих размышлений модель может решить, что информации недостаточно или она неполная, и тогда цикл повторяется: она выполняет новый уточняющий поиск, снова читает и снова думает.

Этот процесс может повторяться несколько раз, пока модель не соберёт достаточно данных и не будет готова предоставить финальный ответ.

### Почему это отличается от API?

*   **В API**: Вы напрямую вызываете одну модель. Когда вы включаете режим мышления (`thinking`), она использует свою внутреннюю логику для рассуждений, но не может взаимодействовать с внешними инструментами, такими как поиск. Это фундаментальное ограничение архитектуры, которое действует на уровне API.
*   **На сайте**: Разработчики DeepSeek создали **оркестратор** — «менеджера», который управляет процессом. Этот оркестратор не спрашивает у одной модели всё сразу. Вместо этого он по очереди вызывает разные инструменты:
    *   Инструмент поиска для получения данных из интернета.
    *   Инструмент чтения для анализа конкретных страниц.
    *   Модель `DeepSeek-R1` (или её аналог) в режиме мышления для логического анализа найденной информации.

Вся эта сложная «кухня» скрыта от глаз пользователя, и создаётся впечатление, что ИИ просто одновременно ищет и думает.

### Что это значит для вашего приложения

Поскольку в основе вашего приложения будет официальное API, а не внутренняя архитектура сайта, вы столкнётесь с тем же ограничением: включить в одном API-запросе и `thinking`, и `search` не получится.

Но, как мы и обсуждали, для вашей задачи это и не нужно, потому что функцию поиска вы заменяете своим **«глобальным» контекстом (RAG)**. Ваш процесс будет выглядеть так:

1.  **Поиск (ваша часть)**: Ваше приложение перед отправкой запроса к API выполняет поиск по своей базе данных и собирает все сообщения, отмеченные для глобального доступа.
2.  **Мышление (DeepSeek)**: Вы отправляете один запрос к API DeepSeek **с включённым режимом мышления** (`thinking`), передав в него всю историю диалога и те самые собранные «глобальные» сообщения в качестве контекста. Модель будет рассуждать, опираясь на предоставленные данные.

Таким образом, вы получите аналог связки «поиск + мышление», только с поиском по вашей личной базе знаний, а не по всему интернету.

Это различие важно понимать на будущее, чтобы не пытаться реализовать в API то, что сайт делает с помощью сложной внутренней оркестровки.

## Техническое задание (переработанное) на веб-приложение «AI Chat с глобальной памятью»

### 1. Общее описание
Веб-приложение — чат-интерфейс для работы с DeepSeek. Открывается в мобильном браузере, выглядит и ведёт себя как нативное приложение для смартфона. Все данные (диалоги, сообщения, настройки) хранятся на сервере и не теряются после перезагрузки.

**Ключевая особенность:** пользователь может отметить любое сообщение или целый чат как «Глобальный». Эта информация становится частью **глобальной памяти** — автоматически подставляется во все чаты при каждом новом запросе к ИИ. Таким образом, DeepSeek всегда «знает» важные для пользователя сведения, действуя в едином контексте, собранном вручную.

**Режим мышления DeepSeek («Глубокое мышление»)** — включён **постоянно** для всех запросов без возможности отключения.

**Внешний поиск в интернете не используется.** Вся «поисковая» функция построена исключительно на описанной выше глобальной памяти.

- **Глобальный чат** — чат, целиком отмеченный как глобальный. Все его сообщения по умолчанию включаются в глобальную память, если у конкретного сообщения этот признак не снят вручную.
- **Глобальная память** — совокупность всех глобальных сообщений из всех чатов. Её содержимое автоматически добавляется в начало любого диалога с ИИ перед отправкой запроса.

#### 3.3. Управление глобальной памятью на уровне сообщений

- Если иконка залитая/цветная — сообщение входит в глобальную память.

Нажатие на иконку переключает состояние для этого конкретного сообщения, **независимо от флага глобальности всего чата**. Таким образом, даже внутри глобального чата можно исключить отдельные сообщения, сняв с них метку, или наоборот — добавить в глобальную память только избранные сообщения обычного чата.

### 4. Логика работы глобальной памяти
Перед отправкой любого запроса в DeepSeek приложение автоматически собирает все сообщения, отмеченные как глобальные, из всех чатов.

Таким образом, ИИ во всех чатах оперирует одними и теми же важными фактами, указанными пользователем. Изменение набора глобальных сообщений мгновенно влияет на все последующие ответы.

1. Собирает **все сообщения** из всех чатов, у которых `is_global = true`, сортирует их по времени создания.
2. Берёт историю текущего чата (сообщения до текущего момента).
3. Формирует финальный массив для API: сначала глобальные сообщения, затем история чата, затем новый запрос пользователя.
4. Добавляет параметр `thinking: enabled`, **если в данном чате включено глубокое мышление**.
5. Отправляет запрос к DeepSeek и стримит ответ обратно в интерфейс.

### 5. Поведение при ошибках
- При обрыве сети или ошибке API — сообщение об ошибке с кнопкой «Повторить».
- При остановке генерации ответ не считается ошибочным, он фиксируется.
- Все изменения (отметки, настройки мышления, переписывание) сохраняются на сервере мгновенно.

### 6. Интерфейс (адаптация под смартфон)
- Минимальный размер интерактивных элементов — 44×44 точки.
- Шрифт сообщений комфортный для чтения без масштабирования.
- Панель ввода всегда видна над клавиатурой.
- Жесты: свайп для удаления чата, долгое нажатие для контекстного меню.
- Иконки глобальности и мышления — компактные, ненавязчивые, интуитивно понятные.
- Цвета: сообщения пользователя и ИИ контрастны друг другу.

### 7. Хранение и безопасность
- Данные в БД на сервере (сообщения, чаты, настройка мышления для чата).
- API-ключ DeepSeek хранится на сервере, в браузер не передаётся.
- HTTPS-соединение.
- На старте однопользовательский режим.
