# Техническое задание: Концепция памяти и наследия мира Экзис

## 1. Анализ задачи и схема взаимодействия

**Цель** — создать двухконтурную систему памяти, которая позволит ИИ-Мастеру (deepseek-v4-pro) сохранять, индексировать и извлекать знания о мире, а человеку – видеть нарративную хронику событий. Система будет работать в паре со средой разработки VSCode + Cline, где Cline выступает в роли агента-исполнителя: он инициирует вызовы API, управляет файловой системой и БД, применяет FIM для точечных правок кода.

**Участники взаимодействия:**
- **Пользователь (Инженер)** – ставит задачи через Cline, просматривает хронику.
- **Cline (агент)** – получает промты, управляет файлами, вызывает функции Python для работы с памятью.
- **DeepSeek API** – предоставляет две ключевые возможности:
  - *Chat Completions* (включая JSON mode) для генерации нарратива, структурирования событий, саммаризации.
  - *FIM (Fill-In-the-Middle)* для вставки кода в существующие файлы без перезаписи.
- **Векторное хранилище** (ChromaDB/FAISS) – быстрый семантический поиск по «LLM-памяти».
- **Реляционная БД** (SQLite/PostgreSQL) – метаданные для человека, связи между сущностями, человекочитаемые ссылки.
- **Файловая система** – папка `chronicles` с художественными Markdown-файлами.

**Принципиальная схема потоков данных:**
```
[Инженер]──(задача)──▶ [Cline] ──(промт + вызовы API)──▶ [DeepSeek API]
                           │                                      │
                           ▼                                      ▼
                   [Управление файлами]               [Генерация текста/JSON]
                           │                                      │
                           ▼                                      ▼
                  [chronicles/*.md]               [Структурированное событие]
                  [код проекта .py]                        │
                           ▲                               ▼
                           │                    [Векторная БД + реляционная БД]
                           └───────────(запись и чтение)──┘
```
**Контур 1 (LLM-память):**  
При возникновении значимого события (диалог, действие, изменение мира) Cline вызывает функцию `save_event`, которая:
1. Отправляет описание в DeepSeek Chat Completions с `response_format={"type": "json_object"}` для приведения к строгой JSON-схеме.
2. Полученный JSON дополняется эмбеддингом (с помощью отдельной модели эмбеддингов) и сохраняется в векторную БД.
3. При необходимости обновляются связанные записи.

При старте новой сессии агент-ритривер:
- Превращает «краткое описание текущего контекста» в эмбеддинг.
- Ищет top-k близких записей.
- Фильтрует по тегам и `related_ids`.
- Внедряет их в системный промт Мастера.

После сессии запускается *Summary Chain*: полный лог сессии отправляется в DeepSeek с инструкцией создать саммари (ограничение по `max_tokens`), которое сохраняется как специальная запись с типом `summary`.

**Контур 2 (Human-память):**  
Параллельно с каждым значимым событием ИИ генерирует художественный пересказ в Markdown, который Cline сохраняет в папку `chronicles/` с именем `session_YYYYMMDD_HHMMSS.md`.  
В реляционной БД хранится таблица `sessions` и таблицы `entities` (NPC, локации, предметы) с перекрёстными ссылками на UUID записей векторного хранилища. Человек может открыть любую хронику и по ссылке «🔍 детали» перейти к технической записи.

```json
              "connected_location_ids": {"type": "array", "items": {"type": "string", "format": "uuid"}},
              "atmosphere": {"type": "string"}
            }
          }
        }
      ]
    }
  }
}
```

**Примечание по эмбеддингам:** т.к. deepseek-v4-pro не предоставляет нативного API эмбеддингов, рекомендуется использовать локальную модель `sentence-transformers` (например, `all-MiniLM-L6-v2`) для офлайн-режима, либо `text-embedding-3-small` от OpenAI с отдельным ключом. Выбор будет сделан в `config.py`.

## 4. Полный алгоритм работы агента-ритривера (псевдокод)

Агент-ритривер встроен в `memory/retriever.py`. Его задача – по текущему запросу (тексту последнего сообщения или краткому контексту) вернуть наиболее релевантные MemoryRecord.

**Алгоритм:**
1. Принять запрос `query_text` (строка).
2. Принять опциональные параметры:
   - `top_k` (по умолчанию 5)
   - `filter_type` (тип записи или список типов)
   - `filter_related_id` (UUID для поиска всех записей, связанных с конкретной сущностью)
   - `filter_tags` (список тегов)
   - `min_importance` (минимальная важность)
3. Получить эмбеддинг `query_embedding = get_embedding(query_text)`.
4. Сформировать метаданные для поиска в ChromaDB:
   - Если заданы фильтры, создать словарь `where = {...}` с условиями `"$and"`.
   - Пример: `{"$and": [{"type": {"$in": ["event", "dialog"]}}, {"importance": {"$gte": 3}}]}`.
5. Выполнить `collection.query(query_embeddings=[query_embedding], n_results=top_k, where=where, include=["metadatas", "documents"])`.
6. Если задан `filter_related_id`, дополнительно найти все записи, где `related_ids` содержит этот UUID (можно сделать второй запрос без эмбеддингов, через чтение всех метаданных, или использовать ChromaDB с индексом по массивам – зависит от версии; для надежности рекомендуется поддержать вручную: в реляционной БД есть таблица `record_relations`, по которой можно получить список id и затем запросить по ним из Chroma).
7. Объединить результаты, отсортировать по релевантности и важности.
8. Вернуть список объектов `MemoryRecord` (без эмбеддингов, только поля для контекста).

**Псевдокод:**
```python
def retrieve(query_text, top_k=5, filters=None):
    emb = get_embedding(query_text)
    where_clause = build_where_clause(filters)
    raw_results = vector_collection.query(query_embeddings=[emb], n_results=top_k, where=where_clause)
    # дополнительные шаги фильтрации по связям, если требуется
    return [MemoryRecord.from_chroma(doc, meta) for doc, meta in zip(raw_results['documents'][0], raw_results['metadatas'][0])]
```

Для экономии токенов ритривер вызывается только в начале сессии и при явной смене контекста (после длинной паузы в диалоге). Текущий буфер диалога хранится в оперативной памяти как список сообщений.

## 5. Код функций на Python

### 5.1 Сохранение события в память

Функция `save_event_to_memory` демонстрирует использование Chat Completions с JSON mode для структурирования и последующего сохранения.

```python
import openai
import uuid
from datetime import datetime, timezone
from memory.config import DEEPSEEK_API_KEY, DEEPSEEK_BASE_URL
from memory.embeddings import get_embedding
from memory.vector_store import add_record

client = openai.OpenAI(api_key=DEEPSEEK_API_KEY, base_url=DEEPSEEK_BASE_URL)

def save_event_to_memory(raw_description: str, event_type: str = "event"):
    """
    Принимает описание события на естественном языке,
    получает от LLM структурированный JSON по схеме MemoryRecord,
    добавляет эмбеддинг и сохраняет в векторную БД.
    Возвращает созданную запись.
    """
    system_prompt = """
    Ты - ассистент Мастера мира Экзис. Преобразуй описание события в строгий JSON, соответствующий схеме MemoryRecord.
    Всегда возвращай только JSON-объект. Поля id, timestamp, embedding будут добавлены позже.
    Для поля related_ids, если нет явных связей, оставь пустой массив.
    Теги должны быть ключевыми словами из описания.
    Определи тип события: event, npc_state, item, quest, location, dialog, world_lore.
    Дополнительные атрибуты помести в metadata согласно типу.
    """
    response = client.chat.completions.create(
        model="deepseek-chat",  # или deepseek-v4-pro при доступности
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": raw_description}
        ],
        response_format={"type": "json_object"},
        temperature=0.3,
        max_tokens=800
    )
    record_json = response.choices[0].message.content
    record = json.loads(record_json)

    # Дополняем обязательные поля
    record["id"] = str(uuid.uuid4())
    record["timestamp"] = datetime.now(timezone.utc).isoformat()
    record["embedding"] = get_embedding(record["description"])
    if "importance" not in record:
        record["importance"] = 5

    # Сохраняем в ChromaDB
    add_record(record)
    return record
```

### 5.2 FIM-вставка кода

DeepSeek предоставляет экспериментальный endpoint `v1/completions` с поддержкой FIM через поля `prompt` и `suffix`. Пример для вставки новой функции в середину Python-файла.

```python
import openai

FIM_BASE_URL = "https://api.deepseek.com/beta"  # специальный base_url для FIM

def fim_insert_code(file_path: str, insert_after_line: int, directive: str):
    """
    Читает файл, разделяет его на prefix (до указанной строки) и suffix (после).
    Отправляет запрос к FIM API DeepSeek с инструкцией directive.
    Полученный код вставляет между частями.
    """
    with open(file_path, 'r', encoding='utf-8') as f:
        lines = f.readlines()

    prefix_lines = lines[:insert_after_line]
    suffix_lines = lines[insert_after_line:]

    prefix = "".join(prefix_lines)
    suffix = "".join(suffix_lines)

    client = openai.OpenAI(api_key=DEEPSEEK_API_KEY, base_url=FIM_BASE_URL)

    response = client.completions.create(
        model="deepseek-chat",  # или deepseek-v4-pro
        prompt=prefix,
        suffix=suffix,
        # подсказка о том, что нужно вставить
        extra_body={
            "instruction": directive
        },
        max_tokens=500,
        temperature=0.2,
        stop=["<|fim_end|>"]  # стоп-токен для FIM
    )
    generated_middle = response.choices[0].text.strip()

    # Собираем новый контент
    new_content = prefix + generated_middle + suffix
    with open(file_path, 'w', encoding='utf-8') as f:
        f.write(new_content)

    return generated_middle
```

**Пояснение:** В реальном API DeepSeek FIM может требовать специального токена `"<|fim_hole|>"` в промпте, но выше показан концептуальный вариант; при реализации нужно уточнить актуальный формат. Параметр `extra_body` с `instruction` – гипотетический; DeepSeek может использовать `stop=["<|fim_end|>"]` и промпт, заканчивающийся на `"<|fim_hole|>"`. Cline адаптирует вызов под документацию.

### 5.3 Вспомогательная функция эмбеддинга (локально)

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
```
