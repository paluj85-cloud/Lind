**ИСКОРЕНЕНИЕ ЕРЕСИ (КЛЮЧЕВЫЕ ИСПРАВЛЕНИЯ):**
В предыдущей версии инструкции вкрались пять ересей (расхождений с ФТЗ). Изгони их без жалости:

1. **Имя поля для имени игрока:** В эндпоинте `POST /api/session/name` тело запроса должно содержать поле `"player_name"`, а не `"name"`. Ответ должен содержать поле `"master_says"`, а не `"message"`.
2. **Протокол WebSocket:** Клиент отправляет строго `{"action": "player_message", "text": "..."}`. Сервер отвечает строго `{"event": "master_thinking", "status": "started"}`, `{"event": "master_speech", "text": "...", "turn_id": "uuid"}`, `{"event": "master_thinking", "status": "stopped"}`. Никаких `"type": "action"` или `"type": "master_message"`.
3. **Приветствие старта сессии:** Ответ `POST /api/session/start` содержит поле `"master_says"` с текстом приветствия и запросом имени. Это нужно проверять.
4. **Отсутствующий эндпоинт:** Эндпоинта `GET /api/session/{id}/history` в ФТЗ нет. Вместо этого для проверки сохранности сессии после перезапуска сервера нужно использовать следующий ритуал: после перезапуска подключиться по WebSocket к тому же session_id и убедиться, что сервер отправляет событие с недавней историей (или, если история не отправляется автоматически, проверить, что `GET /api/session/{id}` возвращает валидные данные сессии, а следующий ход в чате не вызывает ошибки). Опиши оба варианта с пояснением.
5. **Скрипты:** Прилагаемый Python-скрипт для тестирования WebSocket должен использовать правильные имена полей и учитывать новый протокол.
