## А.2 Тестирование WebSocket-чата

Напиши и запусти Python-скрипт `test_chat.py`. Он выполнит полный цикл: создаст сессию, задаст имя, подключится по WebSocket, отправит 3-4 сообщения, выведет в консоль все полученные ответы и проверит наличие индикатора размышлений.

```python
#!/usr/bin/env python3
"""
Скрипт автоматизированной проверки WebSocket-чата «Экзис».
Требуется установка: pip install websockets httpx
"""
import asyncio
import json
import httpx
import websockets

BASE_URL = "http://localhost:8000"
WS_URL = "ws://localhost:8000/ws/chat"   # Базовый путь WebSocket

async def main():
    async with httpx.AsyncClient() as client:
        # 1. Старт сессии
        resp = await client.post(f"{BASE_URL}/api/session/start")
        resp.raise_for_status()
        data = resp.json()
        session_id = data["session_id"]
        print(f"Сессия создана: {session_id}")

        # 2. Установка имени
        resp = await client.post(f"{BASE_URL}/api/session/name", json={
            "session_id": session_id,
            "name": "Альдрик"
        })
        resp.raise_for_status()
        blessing = resp.json()
        print("Благословение:", blessing.get("message", ""))

        # 3. Подключение по WebSocket
        ws_full_url = f"{WS_URL}/{session_id}"
        async with websockets.connect(ws_full_url) as ws:
            print("WebSocket-соединение установлено.")

            # При подключении сервер может сразу отправить историю или приветствие
            # Читаем первое сообщение
            initial = await ws.recv()
            print("Первое сообщение от Мастера:", initial)

            # Функция для получения сообщений с анализом thinking
            async def receive_until_message():
                while True:
                    msg = await ws.recv()
                    data = json.loads(msg)
                    msg_type = data.get("type")
                    if msg_type == "thinking":
                        print("[Индикатор] Мастер размышляет...")
                    elif msg_type == "master_message":
                        return data.get("content", "")
                    else:
                        print(f"Получено неизвестное сообщение: {data}")

            # 4. Отправка нескольких действий игрока
            actions = [
                "осматриваю таверну",
                "спрашиваю у трактирщика про работу",
                "выхожу на улицу",
                "поднимаю голову к небу и зову Экс-Иса"
            ]

            for action in actions:
                print(f"\n>>> Игрок: {action}")
                await ws.send(json.dumps({"type": "action", "content": action}))
                # Ожидаем ответа Мастера, пропуская thinking
                response = await receive_until_message()
                print(f"<<< Мастер: {response}")

            # 5. Закрытие соединения
            await ws.close()
            print("\nWebSocket-соединение закрыто.")

    print("=== Проверка чата завершена ===")

if __name__ == "__main__":
    asyncio.run(main())
```

**Запуск:**
```bash
python3 test_chat.py
```

**Ожидаемый результат:** для каждого действия игрока выводятся (`<<< Мастер: ...`) содержательные ответы, стилизованные под божество. Между отправкой действия и ответом обязательно присутствует сообщение `[Индикатор] Мастер размышляет...`. Если thinking не появляется или ответы пустые/ошибочные – тест провален.

**Ожидаемый результат:** данные точно такие же, как до перезапуска (порядок, содержимое). Сессия не должна быть потеряна. Если возвращается ошибка 404 или пустой массив – провал.

# Раздел Б. Ручная проверка (для Творца)

Данный сценарий предназначен для живого тестирования в браузере. Он дублирует автоматическую проверку, но позволяет оценить визуальные аспекты и удобство использования.

## Б.1 Пошаговый сценарий

### Шаг 1. Открытие главной страницы
- Открой браузер и перейди по адресу `http://localhost:8000` (или актуальному URL сервера).
- **Ожидаемый результат:** отображается приветственное сообщение, поле для ввода имени и кнопка «Войти» или аналогичная.

### Шаг 2. Ввод имени
- Введи имя «Альдрик» (или любое другое) и нажми кнопку подтверждения.
- **Ожидаемый результат:** появляется персональное благословение, стилизованное под голос божества (например, золотистый текст, анимация). Интерфейс переходит в режим чата.

### Шаг 3. Общение в чате
- В поле ввода чата напечатай первое действие: **«осматриваю таверну»**, нажми Enter или кнопку отправки.
- **Ожидаемый результат:**
  - Появляется индикатор «Мастер размышляет…» (например, пульсирующие точки, анимированная иконка).
  - Через 1–3 секунды индикатор исчезает, и появляется ответ Мастера, написанный в художественном божественном стиле (архаичные обороты, образы).
- Отправь ещё сообщения:
  - **«спрашиваю у трактирщика про работу»**
  - **«выхожу на улицу»**
  - **«поднимаю голову к небу и зову Экс-Иса»**
- После каждого сообщения проверяй:
  - индикатор размышления появляется и исчезает;
  - ответы Мастера согласованы по сюжету (например, после выхода на улицу описывается рыночная площадь или небо).
  - сообщения не дублируются, порядок хронологический.

### Шаг 4. Проверка сохранения истории после перезагрузки
- Не закрывая сессию, нажми в браузере кнопку обновления страницы (F5 или Ctrl+R).
- **Ожидаемый результат:** страница должна восстановить ваш чат, отобразив все предыдущие сообщения. Возможно, потребуется повторно ввести имя? Уточни, сохраняется ли сессия автоматически (по кукам) или нужен ручной ввод ID. В идеале сессия восстанавливается без дополнительных действий.
- Если диалог исчез – это серьёзный баг.

### Шаг 5. Проверка адаптивности на мобильных
- Медленно уменьшай ширину окна браузера до минимальной (имитация смартфона).
- **Ожидаемый результат:** интерфейс адаптируется: поле ввода и кнопки не вылезают за границы, текст читаем, чат не ломается. Если есть боковая панель, она должна перестроиться или скрыться.

## Б.2 Использование инструментов разработчика (F12)

На любом этапе ты можешь открыть DevTools (F12) для более глубокой проверки.

### Проверка WebSocket-соединения
1. Перейди на вкладку **Network** (Сеть).
2. В фильтре выбери **WS** (WebSocket).
3. Найди соединение с адресом `/ws/chat/...`.
4. Кликни на него, откроется панель **Messages** (Сообщения).
5. Отправь действие в чате и наблюдай за кадрами:
   - **Зеленые стрелки** (отправлено) содержат JSON типа `{"type":"action","content":"осматриваю таверну"}`.
   - **Красные стрелки** (получено) должны сначала показать `{"type":"thinking","status":true}`, затем `{"type":"master_message","content":"..."}` (иногда с индикатором `thinking: false`).
6. Если после отправки действия нет ответных кадров, или приходит ошибка – соединение нарушено.

### Проверка консоли на ошибки
- Перейди на вкладку **Console** (Консоль).
- **Ожидаемый результат:** после загрузки страницы и всех действий не должно быть красных сообщений об ошибках JavaScript или сетевых сбоях. Возможны предупреждения, но не критические ошибки.

## Б.3 Ожидаемые результаты ручной проверки
- Все шаги сценария выполняются без зависаний и ошибок.
- Индикатор «Мастер размышляет» визуально отображается и исчезает перед каждым ответом.
- Текст ответов Мастера увлекателен, не повторяется шаблонно, выдержан в едином стиле.
- История сохраняется при обновлении страницы.
- Верстка адаптируется под мобильные экраны.

Если хотя бы одно из этих требований не выполнено – заводится отчёт о дефекте с приложением скриншотов/логов консоли.
